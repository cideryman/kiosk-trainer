# P42 게스트 행사명 텍스트 부분 서식(색상/굵기) 툴바 및 레이아웃 보호 워크스루

## 개요

주방 화면의 게스트 행사명 입력창에 **글자 색상 및 굵기(Bold)** 서식 지정 툴바를 추가하고, 손님 앱에서 서식이 적용된 타이틀을 안전하게 렌더링하도록 구현했습니다. 글자 크기 `24px` 고정, 1줄 제한(최대 20자)으로 레이아웃이 100% 일정하게 유지됩니다.

---

## 주요 변경 내역

### 1. 주방 서식 툴바 및 20자 제한 (`kitchen.html`, `js/kitchen.js`)
- `🎯 게스트 운영 메뉴 모드` 서브 패널에 **굵게(`B`)**, **시그니처 6색 팔레트(레드/블루/보라/초록/주황/기본)**, **서식 초기화** 버튼 툴바 추가
- 입력창을 `<input>` → `contenteditable` div로 전환하고 드래그 선택 영역에 `execCommand` 서식이 바인딩되도록 `formatEventText()` 함수 구현
- 최대 20자 실시간 글자 수 카운터(`0/20자`) 표시 및 초과 시 빨간색 경고
- `saveGuestSettings()`에서 `inputEventName.innerHTML.trim()`으로 서식 포함 HTML 문자열을 GAS에 전달
- `loadGuestOpsPanel()`에서 `inputEventName.innerHTML = data.guestEventName`으로 서식 포함 HTML 복원

### 2. 손님 앱 안전 렌더링 (`js/guest.js`)
- `sanitizeEventTitleHtml()` 헬퍼 추가:
  - `&lt;`/`&gt;` HTML 엔티티가 포함된 경우 `DOMParser`로 원본 HTML 태그로 디코딩
  - 허용 태그(`span`, `font`, `b`, `strong`, `i`, `em`, `div`)만 통과, 나머지는 textContent로 치환
  - `<font color="...">` 어트리뷰트와 `style.color` 모두 감지하여 `style.color`로 통합 변환
  - `script`, `iframe` 등 위험 태그 자동 제거
- `updateGuestAppTitle()`에서 `brandTitleEl.innerHTML = sanitizeEventTitleHtml(eventName)` 사용
- 타이틀 영역에 `font-size: 24px; white-space: nowrap; overflow: hidden; text-overflow: ellipsis;` 적용

### 3. 🎉 이모지 고정 접두사 제거
- 기존에 하드코딩되어 있던 `🎉 ` 접두사를 제거하여 주방에서 입력한 텍스트 그대로 표시

## 버그 수정

### 중복 `updateGuestAppTitle` 함수 문제 (근본 원인)
- **증상**: 주방에서 서식을 지정하면 손님 앱에서 `<font color="#e11d48">장애인식</font>` 같은 HTML 코드가 텍스트로 그대로 노출
- **원인**: `js/guest.js`에 `updateGuestAppTitle` 함수가 **2개 중복 정의**되어 있었음. 505행의 새 버전(`innerHTML`)이 543행의 옛 버전(`textContent`)에 의해 덮어써져 항상 `textContent`가 실행됨
- **해결**: 543행의 중복 옛 함수를 삭제하여 `innerHTML` + `sanitizeEventTitleHtml()` 버전만 유지
