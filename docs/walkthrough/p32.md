# P32 구현 완료 워크스루 (Walkthrough)

이 문서에는 **P32 (게스트 주문 페이지 guest.html 내 인라인 CSS/JS 외부 파일 분리 리팩토링)**의 구현 완료 내역과 검증 결과를 기록합니다.

---

## 🌟 구현 핵심 변경 사항

### 1. CSS 파일 추출 및 적용
- [guest.html](../../guest.html) 내부의 게스트 로그인 UI, 단계별 양식 레이아웃 및 반응형 스타일 선언부(기존 16행~207행)를 [css/guest.css](../../css/guest.css) 파일로 안전하게 분리했습니다.
- `guest.html` 상단에 `<link rel="stylesheet" href="css/guest.css">`를 선언해 분리한 스타일시트를 불러오도록 설정했습니다.

### 2. JS 비즈니스 로직 추출 및 적용
- `guest.html` 내부의 카카오 로그인 처리, 신규 이용 신청 접수 폼 검증, 크레딧 한도 조회, 장바구니 메뉴 담기 등 게스트 주문 핵심 비즈니스 로직(기존 491행~2008행)을 [js/guest.js](../../js/guest.js) 파일로 온전히 분리했습니다.
- `guest.html` 하단에 `<script src="js/guest.js"></script>` 참조 구조를 적용했습니다.
- 이를 통해 `guest.html` 파일의 크기는 기존 **2,012라인에서 300라인으로 크게 축소**되었습니다 (코드 용량 약 85% 감소).

### 3. 정적 캐싱 자원 갱신 및 서비스 워커 연동
- 신규 자원인 `css/guest.css`와 `js/guest.js`가 오프라인 환경에서도 안전하게 캐싱되어 동작하도록 [service-worker.js](../../service-worker.js)의 캐싱 리소스 목록(`urlsToCache`)에 추가했습니다.
- 실환경 클라이언트 기기에서의 즉각적인 캐시 갱신 반영을 유도하기 위해 캐시 버전(`CACHE_NAME`)을 `kiosk-cache-v217`에서 `kiosk-cache-v218`로 상향 완료했습니다.
