# P22 구현 완료 워크스루 (Walkthrough)

이 문서에는 **P22 (발달장애인 AAC 답글 작성 UX 및 메인/주문 내역 소통 노출 기능)**의 완료 결과와 구체적인 구현 변경 사항을 요약합니다.

---

## 🌟 구현 핵심 기능 요약

### 1. P22-1: 발달장애인 AAC 그림 답글 작성 UX (`reviews.html`)
- **TTS 자동 음성 안내**: AAC 그림 이모티콘 칩(버튼) 클릭 시 `Web Speech API (SpeechSynthesis)`를 통해 해당 문구가 정갈한 한국어 음성으로 재생되어 근무자가 청각적으로 확인.
- **원터치 즉시 등록**: 별도 [답글 저장] 클릭 없이 이모티콘 클릭 시 즉시 백엔드로 답글 저장.
- **문구 스위칭(교체)**: 다른 이모티콘을 누르면 문구가 덧붙여지지 않고 선택한 이모티콘/문구로 깔끔하게 교체(Switch).
- **시각적 강조 피드백**: 현재 선택된 이모티콘 카드에 **초록색 강조 테두리(Highlight Ring & Border, `.btn-reply-preset.active`)** 적용.
- **중복 제출 방지 (Locking)**: 답글 저장 처리 중 연타로 인한 중복 제출을 막기 위해 `isSubmittingAACReply` 락 및 투명도 비활성화 적용.

---

### 2. P22-2: 메인 화면 & 내 주문 확인 소통 노출 UI

#### 1) 메인 화면 소통 롤링 피드 (`guest.html`)
- **"💌 마음이 따뜻해지는 소통 피드"** 위젯을 메인 화면 카카오 로그인 바 아래에 배치.
- 4.5초 간격으로 `[날짜] [게스트 닉네임] [발달장애인 직원의 AAC 답글 (이모티콘+문구)]` 메시지가 부드럽게 스르륵 전환(Fade Rolling).

#### 2) 내 주문 확인 내역 하단 답글 박스 (`guest-orders.html`)
- 게스트의 내 주문 내역 카드 하단에 내가 작성한 응원 문구와 발달장애인 직원의 **감사 답글 박스**가 표시되어 따뜻한 감동 전달.
