# P24 장애인식 개선 캠페인 및 행사용 간식 스위칭 기능 워크스루

주방에서 클릭 한 번으로 **`🛵 배달왔삼 (기본 모드)`** 과 **`🎉 행사/캠페인 모드 (커스텀 행사명)`** 를 스위칭하고, 관리자에서 간식 제공 대상을 다중 체크박스(`[v] 회원 [v] 게스트 [v] 행사`)로 유연하게 설정할 수 있도록 구현했습니다.

---

## 🛠️ 주요 변경 내역

### 1. 주방 모드 스위칭 UI (`kitchen.html`, `js/kitchen.js`)
- 게스트 운영 설정 내 **게스트 메뉴 모드** 선택 드롭다운(`select#select-guest-menu-mode`) 및 커스텀 행사명 입력 필드(`input#input-guest-event-name`)를 넉넉하고 시원한 UI(높이 44px, 글자 16px)로 구성했습니다.
- 별도의 겉도는 [모드 적용] 버튼을 제거하고, 아래 **[설정 저장]** 버튼을 누르면 `guestBaseCredit`, `guestDeliveryFee`, `todayDeliveryTeam...`, `guestMenuMode`, `guestEventName`이 원스톱으로 저장되도록 개편했습니다.

### 2. 관리자 간식 다중 타겟 및 필터링 (`admin.html`, `js/admin.js`)
- 신규 간식 등록 및 수정 모달의 판매 대상을 **다중 선택 체크박스**(`[v] 회원용 [v] 게스트용 [v] 행사용`)로 개편했습니다.
- 간식 목록 상단에 **[ 🔽 대상별 필터: 전체 | 회원용 | 게스트용 | 행사용 ]** 드롭다운을 추가하여 원하는 타겟의 간식만 한눈에 정렬/조회 가능합니다.

### 3. 백엔드 DB & 다중 타겟 보존 로직 (`gas/21_Snacks.gs`, `gas/60_Settings.gs`)
- `cleanSnackTarget()` 헬퍼 함수를 추가하여 다중 선택된 타겟(`user,guest,event`)이 구글 시트 `간식목록` H열에 유실 없이 정확히 저장되도록 보완했습니다. (기존에 단일 "guest"가 아닐 경우 "user"로 강제 리셋되던 버그 완벽 수정)
- `SHEET.SETTINGS`에 `guestMenuMode` (`'normal'` | `'event'`) 및 `guestEventName` (기본값: `'장애인식 개선 캠페인'`) 저장 및 조회 연동을 완료했습니다.

### 4. 손님 앱 메인 & 스크립트 에러 보완 (`guest.html`, `js/guest.js`)
- `guestMenuMode === 'event'` 적용 시 손님 앱 상단 타이틀이 **`🎉 장애인식 개선 캠페인`** (설정된 커스텀 행사명)으로 동적 전환되도록 연동했습니다.
- `updateGuestAppTitle()` 내 미정의 `esc()` 예외로 인한 `ReferenceError`를 수정하여, 행사 모드 전환 시 손님 앱이 마감 안내로 오작동하던 현상을 완전히 해결했습니다.
