# P40 구현 완료 워크스루 (Walkthrough)

이 문서에는 **P40 (내 주문 확인 guest-orders.html 화면 내 지난 보관 주문 더보기 Lazy Loading)**의 구현 완료 내역과 검증 결과를 기록합니다.

---

## 🌟 구현 핵심 변경 사항

### 1. 백엔드 GAS API (`gas/40_Orders.gs`)
- `getGuestOrderByToken(data)` 및 `getGuestOrdersByGuestKey(data)` API가 `data.includeArchived === true` 파라미터를 수용하도록 업데이트했습니다.
- 기본 `주문내역`(`SHEET.ORDERS`) 시트와 별개로, `includeArchived` 파라미터가 명시될 때만 `주문보관`(`SHEET.ARCHIVE`) 시트를 추가 스캔하여 일치하는 지난 보관 주문을 반환합니다.

### 2. Mock 엔진 (`js/config.js`)
- 개발 및 로컬 테스트 환경에서도 `includeArchived: true` 옵션 요청 시 `localStorage.getItem('mockArchivedOrders')`의 아카이브 데이터까지 함께 검색하도록 대응 조치했습니다.

### 3. 내 주문 확인 화면 (`guest-orders.html`)
- 화면 접속 시에는 당일/진행 중 주문만 **0초 딜레이로 초고속 렌더링**하여 주방 서버 부담과 화면 갱신 지연을 차단했습니다.
- 주문 내역 하단에 `📁 지난 보관 주문 내역 불러오기` 버튼을 배치했습니다.
- 버튼 클릭 시 `includeArchived: true` 파라미터로 데이터를 요청하여 중복 없이 통합 병합(Merge) 및 최신순 정렬하여 보여줍니다.

### 4. 캐시 버전 상향 (`service-worker.js`)
- 클라이언트 기기의 갱신 반영을 보장하기 위해 `service-worker.js`의 `CACHE_NAME`을 `kiosk-cache-v220`에서 `kiosk-cache-v221`로 상향 조치했습니다.
