## 변경 유형별 기본 원칙

- **문서만 변경**: Markdown 링크, 제목 구조, 최신 상태와 DB 헤더 일치 여부를 확인합니다. GAS 재배포나 서비스 워커 갱신은 필요하지 않습니다.
- **정적 HTML/CSS/JS 변경**: 로컬 화면과 모바일 폭을 확인하고 서비스 워커 캐시 버전 상향 필요성을 검토합니다. GitHub Pages 반영 후 카카오와 PWA 캐시를 확인합니다.
- **GAS 변경**: `node check_syntax.js` 통과 후 같은 Apps Script 프로젝트에 반영하고 새 버전을 배포합니다. 관련 운영 흐름을 한 번에 수동검증합니다.
- **DB 구조 변경**: 최신 백업 확보, 실제 1행 헤더 대조, 운영점검 확인 후 낮은 운영 시간에 수행합니다.

## 완료된 회귀 검증 기준

다음 항목은 운영에서 정상 동작을 확인했습니다. 관련 코드를 바꿀 때 회귀 테스트로 다시 사용합니다.

1. 일반 키오스크와 로컬·카카오 게스트의 정상 주문, 크레딧 차감, 주방 접수
2. 주문 버튼 연속 클릭 방어와 단일 주문번호 생성
3. 주문 토큰 기반 조회·취소·후기·사진 업로드 권한 검사
4. 주방 일반·포장·배달 신규 주문 알림음과 화면 필터 독립 동작
5. 제공 상태 Y/N 변경, `updateOrderServed()` 락과 관리자 로그
6. `getSnacks` 15초 캐시, 주문·취소·관리자 수정 뒤 재고와 판매 상태 반영
7. Pretendard 웹폰트 요청 제거와 시스템 폰트 fallback
8. 관리자 이용자·간식 이미지 최대 800px WebP 업로드
9. 카카오 이용자의 `(비회원)` 꼬리 제거가 주방·전광판·주문조회·인쇄에 반영
10. 주문내역 A:R에서 주문보관 A:R로 열 밀림 없는 아카이빙
11. 관리자·주방·후기 상단 하위 메뉴의 배타 동작과 바깥 클릭·`Esc` 닫기
12. 배달왔삼 마감 중 읽기 전용 메뉴와 예약 종료 주문 5분 유예의 로컬 경계값

## 현재 배포 확인 항목

1. 관리자·주방·후기 화면 상단에 관리자 잠금 해제 폼이 표시됩니다.
2. 잘못된 관리자 키는 저장되지 않고, 올바른 키만 `sessionStorage`에 저장됩니다.
3. `잠그기`를 누르면 키가 삭제되고 보호된 작업을 다시 실행할 때 입력을 요구합니다.
4. 관리자 권한 거절 응답 후 키가 삭제되고 상단 입력 폼으로 돌아옵니다.
5. 관리자 키를 입력한 POST 요청이 GAS 실행 로그에 남더라도 키 값은 기록되지 않습니다.
6. 직원 전용 기기에서만 브라우저 비밀번호 저장을 선택하고, 공용 키오스크에서는 저장하지 않습니다.
7. 관리 화면의 `외부 화면 > 배달왔삼`이 실제 `guest.html`을 엽니다.
8. 영업 중에는 정상 주문 흐름이 보이고, 마감 상태에서는 `메뉴 미리보기`만 제공됩니다.
9. 읽기 전용 메뉴에는 수량 조절, 장바구니, 주문 확인 버튼이 없습니다.
10. 예약 종료 전 시작한 주문은 종료 후 5분 안에 접수됩니다.
11. 즉시 마감 후 제출과 예약 종료 5분 이후 제출은 거절됩니다.

## 선택 심화 검증

같은 사용자와 동일한 `idempotencyKey`로 같은 `placeOrder` 요청을 강제로 두 번 보낼 수 있을 때 주문 행, 재고, 크레딧이 한 번만 변경되고 두 번째 응답이 기존 `orderNo`와 `orderToken`을 반환하는지 확인합니다.
