## 2026-07-10 - 주방 재고 및 외부 화면 버튼 검토 등록

- 주방 간식 재고 현황 복구를 P3으로 등록했습니다. 게스트·일반 그룹, 간식명·재고만 표시, 재고 0 하이라이트를 기준으로 검토합니다.
- 외부 화면 빠른 버튼 시각화 실험을 P4로 등록했습니다. 일반 키오스크 주황색, 배달왔삼 청록색, 전광판 보라색을 시험하고 시각적 복잡성을 확인합니다.
- 두 항목은 아직 구현하지 않고, 우선순위에 따라 하나씩 설계·구현·수동 검증합니다.

## 2026-07-10 - 관리 화면 전환 탭과 기능 버튼 분리

- 세 화면의 전환 탭을 컨테이너 최상단으로 이동했습니다.
- 기존 색상 헤더의 큰 제목과 설명 문구는 제거하고, 접근성을 위해 숨김 `h1`만 유지했습니다.
- `admin.html`, `kitchen.html`, `reviews.html`의 헤더에서 중복된 화면 전환 버튼을 제거했습니다.
- 컨테이너 최상단에 관리자·주방·후기 전역 탭을 배치하고 화면별 고유 색상, 활성 상태, `aria-current`를 적용했습니다.
- 알림·운영 도구·외부 화면·새로고침 등 현재 화면 기능 버튼은 별도 도구 영역에 유지했습니다.
- 기능 버튼을 데스크톱 헤더 우측으로 이동하고, 탭 영역을 전체 폭으로 확장해 관리자·후기 탭 끝선과 색상 헤더 끝선을 맞췄습니다. 탭과 헤더 사이 하단 회색선을 제거했습니다.
- 데스크톱과 390px 모바일 뷰포트에서 최상단 탭, 기능 버튼 분리, 가로 넘침 없음, 콘솔 오류 없음을 확인했습니다.
- 정적 파일 변경에 맞춰 `service-worker.js` 캐시 버전을 `kiosk-cache-v137`으로 올렸습니다.

## 2026-07-10 - 관리자·후기 화면 1200px 확대 및 주방 화면 1400px 복원 (하이브리드 조율)

- 관리자(`admin.html`)와 후기(`reviews.html`) 화면의 데스크톱 최대 가로폭을 `1200px` (width: 95%)로 상향하여 테이블 정보 가독성을 개선했습니다.
- 주방 화면(`kitchen.html`)은 실시간 주문 관리의 3열 레이아웃 사용성을 보존하기 위해 기존 `1400px` (width: 95%)로 유지(롤백)했습니다.
- 모바일 해상도(767px 이하) 환경에서 가로 넘침 및 반응형 렌더링에 이상이 없음을 확인했습니다.
- 정적 파일 캐시 갱신을 위해 `service-worker.js` 캐시 버전을 `kiosk-cache-v134`로 올렸습니다.

## 2026-07-10 - 핸드오프 문서 분리 및 DB 재검증

- `handoff.md`를 목차와 현재 상태를 담는 유일한 입구로 축소했습니다.
- 현재 작업, 구조, 결정, DB, 검증, 로그, 에셋 문서를 `docs/handoff/` 아래 역할별로 분리했습니다.
- 기존 `handoff.md` 전체는 `handoff-snapshot-2026-07-10.md`로 보존했습니다.
- 업로드된 `주간보호 매점DB (2).xlsx`의 10개 시트와 실제 1행 헤더를 다시 확인했습니다.
- `주문내역` A:W 23열, `주문보관` A:R 18열, `후기내역` A:J 10열, `간식목록` A:I 9열을 최신 구조로 기록했습니다.

## 2026-07-10 - 관리 화면 메뉴와 배달왔삼 마감 흐름

- 관리자·주방·후기 화면의 상단 하위 메뉴를 배타적으로 동작하게 변경했습니다.
- 다른 상단 버튼, 메뉴 선택, 바깥 클릭, `Esc`에서 열린 메뉴가 닫힙니다.
- `배달왔삼 미리보기` 링크를 실제 `배달왔삼` 화면으로 통일했습니다.
- 마감 중에는 `menu.html?browse=guest` 읽기 전용 메뉴를 제공합니다.
- 예약 종료 전에 시작한 주문은 종료 후 5분까지 허용하고 즉시 마감은 바로 거절합니다.
- 정적 캐시 버전을 `kiosk-cache-v130`으로 올렸습니다.

## 2026-07-10 - 관리자 잠금 해제 UI 및 키 로그 보호

- 관리자·주방·후기 화면의 시스템 `prompt()` 입력을 상단 공통 잠금 해제 폼으로 교체했습니다.
- `verifyAdminAccess`로 서버 검증에 성공한 키만 `sessionStorage`에 저장합니다.
- 잠그기 또는 관리자 권한 거절 시 세션 키와 입력값을 삭제합니다.
- `autocomplete="current-password"`와 안정적인 사용자명 힌트를 사용해 브라우저 비밀번호 관리자와 연동할 수 있게 했습니다.
- GAS `doPost()`는 관리자 키가 포함된 전체 요청 본문을 기록하지 않고 action만 기록합니다.
- 서비스 워커 캐시 버전을 `kiosk-cache-v132`로 갱신했습니다.
- GAS 배포 후 운영 수동검증과 데스크톱·모바일 UI 검증을 완료했습니다.

## 2026-07-10 - `placeOrder` 트랜잭션 정합성 검토

- `LockService`와 주문 전 사전 검증은 동시 주문과 정상적인 재고·크레딧 부족을 방어하지만, Google Sheets 다중 쓰기는 원자적 트랜잭션이 아님을 확인했습니다.
- 주문행 저장 후 재고·크레딧을 차감하므로 후속 쓰기 실패 시 부분 성공 위험이 있습니다.
- `idempotencyKey`도 주문행 저장 후 별도 기록되어 키 기록 실패 시 재전송 방어가 약해질 수 있습니다.
- 실제 불일치 사례가 없으므로 대규모 롤백 구조는 보류하고, 부분 실패가 관찰될 때 복구 설계를 우선순위로 올리기로 했습니다.

## 2026-07-10 - 로컬 게스트 표시명 하루 1회 정책

- 로컬 표시명을 날짜와 함께 저장해 같은 날짜에는 재사용하도록 변경했습니다.
- 주문 성공 후에만 표시명을 확정하고, `guest.html`과 `confirm.html`에서 당일 표시명을 읽기 전용으로 유지합니다.
- 카카오 로그인 이용자의 서버 게스트 프로필 저장·수정 흐름은 변경하지 않았습니다.
- 서비스워커 캐시 버전을 `kiosk-cache-v135`로 갱신했습니다.
- 새 방문자의 입력 가능 상태와 정책 UI 로드를 확인했고, 첫 주문·당일 재주문·주문 실패 후 미확정 동작을 수동검증했습니다.
- 다음 날 새 표시명 입력 허용 여부는 날짜 전환 후 추가 확인이 남아 있습니다.

## 2026-07 - 최근 안정화 요약

- `placeOrder`에 W열 `idempotencyKey` 기반 최소 멱등성을 적용했습니다.
- 비회원 게스트 닉네임 입력을 마지막 주문자 정보 영역으로 이동하고 같은 기기의 다음 주문에 표시명을 재사용합니다.
- GAS 단일 파일을 `gas/*.gs` 기능별 파일로 분리하고 운영 검증을 완료했습니다.
- `getSnacks` 15초, 운영설정 30초, 주문 읽기 2초 서버 캐시와 변경 시 무효화를 적용했습니다.
- 관리자 이미지 업로드를 최대 800px WebP로 경량화하고 Pretendard 웹폰트 다운로드를 제거했습니다.
- `updateOrderServed()`에 `LockService`를 적용하고 주문 읽기 범위를 B:I로 줄였습니다.
- 카카오 연동 이용자의 `(비회원)` 표시 재발 문제를 수정하고 운영 화면을 재검증했습니다.
