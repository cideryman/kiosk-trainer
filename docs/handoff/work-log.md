[핸드오프로 돌아가기](../../handoff.md)

이 문서는 최근 완료 작업의 짧은 결과만 유지합니다. 상세 분석과 이전 원문은 [분리 전 핸드오프 스냅샷](handoff-snapshot-2026-07-10.md), 장기 기록은 [핸드오프 아카이브](handoff-archive.md)에 보관합니다.

## 2026-07-11 - 후기 목록 `더 보기` 방식 적용

- 후기 목록을 공개/숨김 필터 적용 후 초기 20개씩 표시하도록 변경했습니다.
- `후기 더 보기`를 누르면 다음 20개를 같은 목록에 추가하며, 카드 원래 인덱스를 유지해 상세 모달의 이전·다음 동작을 보존합니다.
- 기존 `getReviewsForAdmin` GAS API와 후기 데이터 구조는 변경하지 않았습니다.
- 서비스워커 캐시를 `kiosk-cache-v171`로 갱신했습니다.
- **수동 검증 완료**: 사용자가 20개 초과 목록, 더 보기, 필터, 상세·이전·다음, 답글·공개 상태 변경을 확인했습니다.

## 2026-07-11 - 주방 화면 주문·통계·재고 탭 분리

- 주방 화면에 `주문`, `통계`, `재고` 3개 업무 탭을 추가했습니다.
- 주문 탭에는 대기 주문과 제공 완료 목록, 통계 탭에는 간식별·이용자별 집계, 재고 탭에는 현재 간식 재고 현황을 배치했습니다.
- PC·모바일 모두 같은 탭 구조를 사용하고, 요약 카드·게스트 운영 설정·새로고침 제어는 항상 표시합니다.
- 기존 API 조회, 자동 새로고침, 주문 처리와 필터 ID는 변경하지 않았습니다.
- 서비스워커 캐시를 `kiosk-cache-v169`로 갱신했습니다.
- **수동 검증 완료**: 사용자가 PC·390px·320px 탭 전환과 주문·통계·재고 표시, 자동 새로고침 후 선택 탭 유지를 확인했습니다.

## 2026-07-11 - 주방 상단 요약 카드 문구·값 분리

- 요약 카드 제목을 `오늘 누적 크레딧`, `오늘 총 주문`으로 줄였습니다.
- 누적 크레딧은 `🪙 숫자`, 주문 수는 `숫자건`으로 표시해 모바일에서 의미와 숫자를 빠르게 구분하도록 했습니다.
- 기존 2열 카드와 집계 계산은 유지하고, 모바일 제목 한 줄·카드 여백·값 크기만 조정했습니다.
- 서비스워커 캐시를 `kiosk-cache-v170`으로 갱신했습니다.
- **수동 검증 완료**: 사용자가 PC·390px·320px 제목 줄바꿈과 실제 값 갱신을 확인했습니다.

## 2026-07-11 - 주방 요약·업무 탭 및 후기 더 보기 수동 검증 완료

- 사용자가 M-P3 주방 요약 카드, M-P4 주문·통계·재고 탭, P6 후기 더 보기 기능을 모두 수동 검증했습니다.
- 세 작업을 구현 및 수동 검증 완료 상태로 갱신했습니다.

## 2026-07-11 - 게스트 주문 운영 패널 모바일 재배치

- `kitchen.html`의 게스트 운영 패널을 상태 정보, 기본 설정, 오늘의 배달팀, 운영 시간 제어 영역으로 나누고 의미별 클래스를 적용했습니다.
- 모바일에서는 마감 예정·남은 시간을 2열, 기본 크레딧·배달비를 2열, 기본 배달지를 전체 폭으로 표시합니다.
- 배달팀 담당자 입력은 모바일에서 세로로 안정적으로 배치하고, 20분·30분·1시간·마감 버튼은 2x2로 정리했습니다.
- PC 레이아웃과 기존 입력 ID·저장 함수·자동 새로고침 입력 보호 로직은 유지했습니다.
- 모바일에서 기본 크레딧과 배달비 입력칸이 서로 침범하지 않도록 입력 폭·라벨 줄바꿈을 보완하고, 주방 버튼 문구를 `📊 운영 결과`로 줄였습니다.
- 게스트 운영 설정의 기본 크레딧 표시를 `기본 🪙`으로 줄였습니다.
- 배달팀 설정의 `표시하기 타이틀:`을 `표시`·`제목:`으로 줄이고, 직접 입력 열기 영역을 `⏱ 직접 열기:` 한 줄 배치로 보완했습니다.
- 모바일 `표시`·`제목:` 행의 불필요한 여백을 줄여 제목 입력 폭을 확장하고, 메시지 입력 글자 크기를 조정했습니다.
- 서비스워커 캐시를 `kiosk-cache-v168`으로 갱신했습니다.
- **수동 검증 대기**: 390px·320px 모바일과 PC에서 입력·저장·운영 시간 제어 및 줄바꿈을 확인합니다.

## 2026-07-11 - 관리자 전체 화면 로그인 게이트 및 공통 이미지 적용

- `admin.html`, `kitchen.html`, `reviews.html` 접속 시 관리자 키 입력 게이트가 먼저 표시되도록 했습니다.
- 인증 전에는 관리 화면 본문과 초기 데이터 API 조회가 실행되지 않으며, 인증 성공 후 기존 화면을 불러옵니다.
- `sessionStorage` 인증 상태는 같은 탭·PWA 세션에서 유지하고, 잠그기·권한 거절 시 게이트 상태로 되돌립니다.
- 사용자가 제공한 관리자 이미지를 로그인 게이트와 `icons/admin-180.png`, `icons/admin-192.png`, `icons/admin-512.png`에 적용했습니다.
- 세 관리자 매니페스트는 기존 공통 관리자 아이콘 경로를 사용하므로 별도 매니페스트 구조 변경 없이 아이콘을 통일했습니다.
- 로그인 후 기능 행의 마지막에 열쇠 잠금 버튼을 추가하고, 기존 인증 바는 숨기도록 했습니다. 모바일에서는 관리자·주방·후기 화면의 마지막 기능 행을 3열 구조로 맞췄습니다.
- 서비스워커 캐시를 `kiosk-cache-v163`으로 갱신했습니다.
- **배포 후 검증 대기**: 세 관리 URL 직접 접속 시 정보 차단, 로그인·잠금·탭 이동, PC·모바일 게이트 레이아웃, PWA 아이콘 표시를 확인합니다.

## 2026-07-11 - 모바일 개선 상담 과제 우선순위 등록

- 주문보관 오류·중복 방지 작업은 수정 완료 상태로 유지합니다.
- 후속 모바일 개선 과제는 관리자 로그인 게이트(M-P1), 게스트 주문 운영 패널 재배치(M-P2), 주방 요약 카드 문구·값 분리(M-P3) 순서로 등록했습니다.
- 세 과제 모두 이번 기록에서는 구현하지 않고, 우선순위에 따라 하나씩 설계·구현·검증합니다.

## 2026-07-11 - 주문보관 행 수 경고 및 한글 확인 문구 적용

- `getOrdersToday` 응답에 `orderSheetRowCount`를 추가하고, 주방 화면에 100행 이상 `주문보관 권장`, 300행 이상 `주문보관 필요` 경고를 표시하도록 했습니다.
- 경고는 별도 반복 API 호출 없이 기존 주문 조회 응답과 함께 갱신됩니다.

- `보관 점검` 통과 전에는 보관 버튼을 잠그고, 실행 전 확인창과 확인 문구를 모두 요구합니다.
- 기존 `주문보관`은 실행 직전 자동 백업되며, 과거 주문의 열 구조 차이는 최신 23열 기준으로 보존합니다.

## 2026-07-10 - 후기 목록 `더 보기` 방식 우선순위 등록

- 후기 수가 늘어날 때를 대비해 초기 20~30개 표시 후 `더 보기`로 추가하는 방식을 P6 검토 과제로 등록했습니다.
- 최신순, 비공개 후기 필터, 스크롤 위치를 유지하고, 필요 시 GAS API 페이지 조회까지 함께 검토합니다.

## 2026-07-10 - 주방 제공완료 목록 모바일 카드형 개선
