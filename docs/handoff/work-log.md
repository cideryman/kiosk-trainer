## 2026-07-12 - 이용 신청 모집 정원 관리자 조절 추가

- 고정 5명이던 시범 신청 정원을 `운영설정`의 `guestApplicationCapacity`로 옮기고 기본값을 5명으로 유지했습니다.
- 관리자 신청 설정에 `모집 정원` 숫자 입력을 추가해 1~100명 사이로 저장할 수 있게 했습니다.
- 정원을 현재 활성 신청보다 낮추면 기존 신청은 보존하고 신규 접수·정원 초과 재승인만 차단하며, 다시 높이면 운영 토글이 켜진 상태에서 자동 재개합니다.
- 이전 관리자 화면이 정원 값을 보내지 않는 배포 전환 구간에는 현재 설정값을 보존하도록 하위 호환성을 유지했습니다.
- 신청 설정 캐시 키를 `guestApplicationSettings.v3`, 서비스워커 캐시를 `kiosk-cache-v198`로 갱신했습니다.

## 2026-07-12 - 배달왔삼 1차 시범 신청 5명 제한 구현

- `PENDING + APPROVED`를 5명 정원으로 계산하고, `REJECTED + INACTIVE`는 자리를 반환하도록 GAS 서버 판정을 추가했습니다.
- 동일 `requestId` 재전송과 동일 연락처 확인을 정원 판정보다 먼저 수행하고, 행 추가 직전에 `LockService` 안에서 원본 시트 기준 정원을 다시 확인합니다.
- 정원 초과 요청은 `APPLICATION_FULL`로 응답해 개인정보를 저장하지 않으며, 공개 신청 페이지에는 남은 인원과 1차 모집 마감 안내를 표시합니다.
- 관리자 신청 설정에는 `1차 5명 · 접수 N명 · 남음 N명` 현황을 읽기 전용으로 표시하고, 정원이 차도 관리자의 신청 운영 토글 설정값은 유지합니다.
- 반려·중지 신청을 재승인할 때 이미 다른 5명이 정원을 채웠다면 관리자 상태 변경도 `APPLICATION_FULL`로 차단합니다.
- 신청 접수·상태 변경 후 30초 신청 설정 캐시를 즉시 무효화하고 로컬 Mock에도 같은 정원 규칙을 반영했습니다.
- 정원 필드가 없는 이전 캐시 응답과 섞이지 않도록 신청 설정 캐시 키를 `guestApplicationSettings.v2`로 올렸습니다.
- 서비스워커 캐시를 `kiosk-cache-v197`로 갱신했습니다.
