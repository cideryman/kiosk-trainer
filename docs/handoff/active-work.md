## 상담 후 다음 모바일 개선 큐

### M-P1 - 관리자 로그인 게이트 검토·구현 (구현 완료, 배포 후 검증 대기)

- 관리자·주방·후기 화면에서 인증 전 본문과 데이터를 숨기고, 인증 성공 후에만 화면을 표시하는 전체 화면 게이트를 적용했습니다.
- 기존 `AdminAuth`를 재사용하고, 세 화면의 초기 데이터 조회를 인증 성공 이후로 이동했습니다.
- 관리자 이미지를 로그인 게이트와 `admin-180/192/512.png` 공통 PWA 아이콘에 적용했습니다.
- 같은 브라우저 탭·PWA 세션에서는 인증 상태를 유지하고, 잠그기 또는 권한 거절 시 게이트로 돌아갑니다.
- 인증 성공 후 기존 인증 바는 숨기고, PC에서는 기능 버튼의 마지막에 열쇠 버튼을 두며 모바일에서는 세 열 그리드의 마지막 칸에 고정했습니다.
- **배포 후 검증**: 세 URL 직접 접속 시 정보 비표시, 올바른 키·잘못된 키, 탭 이동 후 재인증 여부, PC·모바일 열쇠 위치와 잠그기, 이미지 크기와 PWA 아이콘을 확인합니다.

### M-P2 - 게스트 주문 운영 패널 모바일 재배치 (구현 완료, 수동 검증 대기)

- 상태 정보와 기본 설정을 모바일 2열 그룹으로 묶고, 기본 배달지는 전체 폭으로 배치했습니다.
- 배달팀 설정은 제목·저장, 표시 여부·타이틀, 담당자 입력, 한마디 입력 영역으로 나누고, 운영 시간 버튼은 모바일 2x2 그리드로 정리했습니다.
- 모바일 기본 설정 입력 폭과 라벨 줄바꿈을 조정해 `배달비`가 기본 크레딧 영역을 침범하지 않도록 보완했고, 주방 기능 버튼 문구를 `📊 운영 결과`로 줄였습니다.
- 게스트 운영 설정의 기본 크레딧 라벨을 `기본 🪙`으로 줄여 모바일 표시 폭을 추가로 확보했습니다.
- 배달팀 설정 문구를 `표시`·`제목:`으로 줄이고, 직접 입력 제어를 `⏱ 직접 열기:` 한 줄로 정리했습니다.
- 모바일 `표시`·`제목:` 행을 flex 배치로 바꾸고 제목 입력 폭을 확장했으며, 메시지 입력 글자 크기도 조정했습니다.
- PC에서는 기존 가로 배치를 유지하고 모바일에서만 패널 여백·줄바꿈·입력 폭을 조정했습니다.
- 자동 새로고침 중 입력 중인 설정값을 덮어쓰지 않는 동작을 유지합니다.
- **수동 검증**: 모바일 390px·320px에서 입력 영역, 저장, 20/30/60분 열기, 마감, 직접 입력 열기와 PC 레이아웃 회귀를 확인합니다.

### M-P3 - 주방 상단 요약 카드 문구·값 분리 (구현 및 수동 검증 완료)

- `오늘 누적 크레딧`·`오늘 총 주문`으로 제목을 줄이고, 값은 `🪙 숫자`·`숫자건` 형식으로 다음 줄에 표시합니다.
- 현재 2열 카드 구조와 통계 의미는 유지합니다.
- 모바일에서 제목은 한 줄로 유지하고 카드 안쪽 여백과 값 글자 크기를 조정했습니다.
- **수동 검증 완료**: PC·390px·320px에서 제목 줄바꿈 없음, 실제 누적 크레딧·주문 수 갱신, 탭 전환 후 값 유지를 확인했습니다.

### M-P4 - 주방 화면 주문·통계·재고 탭 분리 (구현 및 수동 검증 완료)

- 주문 탭에는 간식 챙겨주기와 제공 완료 목록을 배치하고, 통계 탭에는 간식별 집계·이용자별 주문 건수를 배치했습니다.
- 재고 탭에는 현재 간식 재고 현황을 배치했습니다.
- PC·모바일 모두 같은 3탭 구조를 사용하며, 오늘 요약 카드·게스트 운영 설정·새로고침 제어는 탭 밖에 유지합니다.
- 기존 조회·자동 새로고침·주문 처리 ID와 렌더링 로직은 유지하고 탭 표시만 전환합니다.
- **수동 검증 완료**: PC·390px·320px에서 세 탭 전환, 주문 처리·필터, 통계·재고 표시, 자동 새로고침 후 탭 유지, 가로 넘침 없음을 확인했습니다.

### 완료 - 주문보관 행 수 경고 및 안전 실행 확인

- `getOrdersToday` 응답에 실제 `주문내역` 데이터 행 수를 포함해 주방 화면에서 운영 시트 규모를 확인할 수 있게 했습니다.
- 100행 이상이면 `주문보관 권장`, 300행 이상이면 `주문보관 필요` 경고를 표시합니다. 경고는 오늘 주문 수가 아니라 `주문내역` 전체 데이터 행 수를 기준으로 합니다.
- `보관 점검`은 시트를 변경하지 않으며, 점검 기준을 통과한 경우에만 `지난 주문 보관` 버튼을 활성화합니다.
- 보관 실행 확인 문구는 영어 문자열 대신 `주문보관확인`을 사용합니다.
- 기존 보관 시트는 실행 직전에 자동 백업하고, 중복 제거·23열 병합·주문내역 정리 로직을 거친 뒤 보관합니다.
- **배포 후 검증 대기**: 99행 이하 숨김, 100행 이상 권장, 300행 이상 필요 표시와 한글 확인 문구를 확인합니다.

## Priority Queue

### 완료 - 관리자 잠금 해제 UI 및 관리자 키 로그 보호

- **목적**: 작업 중 갑자기 나타나는 시스템 `prompt()`를 없애고, 관리자·주방·후기 화면에서 관리자 권한 상태와 입력 위치를 일관되게 제공합니다.
- **선행 보안 작업**: `doPost()`가 현재 POST 본문 전체를 `Logger.log()`에 기록하므로 `adminToken`이 실행 로그에 남지 않도록 제거하거나 마스킹합니다.
- **권장 UI**: 각 관리 화면의 색상 헤더 바로 아래에 얇은 공통 잠금 해제 바를 둡니다. 잠금 상태에서는 비밀번호 입력과 `잠금 해제`, 성공 후에는 상태 배지와 `잠그기`만 표시합니다.
- **검증 방식**: 입력한 키를 서버에서 먼저 확인하고 성공한 값만 `sessionStorage`의 `kioskAdminToken`에 저장합니다. 권한 거절 시 즉시 삭제합니다.
- **저장 원칙**: 앱이 관리자 키를 `localStorage`에 장기 저장하지 않습니다. 표준 비밀번호 입력 폼을 사용해 브라우저 저장 여부는 사용자가 선택하게 합니다.
- **기기 정책**: 직원 전용이고 화면 잠금이 적용된 관리 기기에서는 브라우저 비밀번호 저장을 허용할 수 있습니다. 이용자가 만지는 공용 키오스크에서는 저장하지 않습니다.
- **구조 개선**: 세 HTML에 중복된 `getAdminToken()`, 상태 표시, 권한 오류 초기화 코드를 공통 헬퍼로 묶을지 구현 전 검토합니다.
- **구현 결과**:
  - `js/app.js`에 공통 `AdminAuth` 헬퍼를 추가하고 세 화면의 상단 폼과 연결했습니다.
  - `verifyAdminAccess` 서버 검증 후에만 `sessionStorage`에 키를 저장하고, 잠그기·권한 거절 시 즉시 삭제합니다.
  - `gas/01_Router.gs`는 요청 본문 대신 action만 로그에 기록해 `adminToken`이 GAS 실행 로그에 남지 않도록 했습니다.
  - `service-worker.js` 캐시 버전을 `kiosk-cache-v135`으로 올렸습니다.
  - GAS 배포 후 올바른 키·잘못된 키·잠그기·권한 거절·브라우저 비밀번호 저장 동작을 운영 기기에서 확인했습니다.
  - 데스크톱과 모바일에서 입력창 숨김, 잠금 토글, 버튼 위치 고정이 정상 동작함을 확인했습니다.

### 완료 - 관리자 화면 폭 통일 및 반응형 정리

- **목적**: 주방 화면은 3열 레이아웃 사용성을 보존하기 위해 1400px을 유지하되, 관리자·후기 화면은 1200px로 확대하여 가로 부족 현상을 개선하고 페이지 전환 시의 이질감을 완화합니다.
- **구현 결과**:
  - `admin.html`과 `reviews.html` 내 PC 뷰 최대 가로폭을 `1200px` (width: 95%)로 확대하고, `kitchen.html`은 기존 `1400px` (width: 95%)로 유지(롤백)하는 조율안을 적용했습니다.
  - 모바일 해상도(767px 이하)에서 테이블과 잠금 바 등이 가로로 깨지지 않도록 반응형 구조를 점검했습니다.
  - `service-worker.js` 캐시 버전을 `kiosk-cache-v134`로 상향했습니다.
  - 관리자·후기 화면을 1200px, 주방 화면을 1400px로 유지하는 하이브리드 구성을 적용했습니다.
  - 실제 모바일 환경에서 가로 넘침과 스크롤 작동을 확인했습니다.

### P1 - 비회원 게스트 주문표시명 하루 1회 정책 (구현 완료, 날짜 전환 검증 대기)

- **목적**: 같은 사람이 당일 주문마다 다른 표시명을 사용해 배달·주방·전광판에서 식별이 어려워지는 상황을 줄입니다.
- **구현 결과**: `localGuestDisplayName`을 `{ dateKey, displayName }` 형태로 저장하고, 기존 문자열 저장값은 최초 접근 시 오늘 날짜 형식으로 마이그레이션합니다.
- **구현 결과**: 주문 성공 후에만 로컬 표시명을 확정하며, 같은 날짜에는 `guest.html`과 `confirm.html`의 입력창을 읽기 전용으로 만들고 랜덤 이름 버튼을 숨깁니다.
- **정책 범위**: 이번 변경은 로컬 게스트에만 적용합니다. 카카오 이용자는 기존 GAS 게스트 프로필 저장·수정 흐름을 유지합니다.
- **다음 날 처리**: 날짜가 바뀌면 오래된 로컬 표시명을 자동 폐기하고 새 표시명 입력을 허용합니다.
- **데이터 원칙**: 표시명은 로컬 기기에만 저장하고 배달지, 원본 카카오 식별자, 크레딧 정보는 추가 저장하지 않습니다.
- **수동 검증 완료**: 첫 주문 성공 후 같은 날 재주문에서 입력 고정·동일 표시명 전송, 주문 실패 시 이름 미확정까지 확인했습니다.
- **남은 검증**: 날짜가 바뀐 뒤 오래된 표시명이 자동 폐기되고 새 표시명 입력이 허용되는지 확인합니다.

### 완료된 검토 - `placeOrder` 트랜잭션 정합성 (구조 개편 보류)

- **목적**: 주문 행 기록, 재고 차감, 일반·게스트 크레딧 차감 중간에 오류가 발생했을 때 부분 성공을 탐지하거나 복구할 필요가 있는지 판단합니다.
- **현재 판단**: 지금은 보류합니다. 멱등성은 동일 요청 재전송 방지이고 트랜잭션은 부분 실패 복구이므로 한 작업으로 섞지 않습니다.
- **착수 조건**: 주문 행은 있으나 재고나 크레딧 반영이 다르다는 실제 사례가 생기거나 운영 점검에서 불일치가 확인되면 P1으로 올립니다.
- **금지선**: 증거 없이 대규모 롤백 구조나 별도 원장 시트를 먼저 도입하지 않습니다.

### P2 - 관리 화면 전환 탭과 기능 버튼 분리 (구현 및 수동 검증 완료)

- **목적**: 관리자·주방·후기 화면 전환과 알림·운영 도구·외부 화면 같은 현재 화면 기능을 시각적으로 분리합니다.
- **권장안**: 상단에 관리자·주방·후기 전역 탭을 두고 각 화면의 고유 색상과 활성 상태를 표시합니다. 기능 버튼은 별도 도구 영역에 배치합니다.
- **반응형 기준**: 모바일에서는 탭이 겹치지 않도록 가로 스크롤 또는 균등 배치를 사용하고, 색상만으로 의미를 전달하지 않습니다.
- **구현 결과**: `admin.html`, `kitchen.html`, `reviews.html`의 헤더에서 중복 화면 이동 버튼을 제거하고, 컨테이너 최상단에 관리자·주방·후기 전역 탭을 배치했습니다.
- **구현 결과**: 관리자·주방·후기 탭에 각 화면 고유 색상과 활성 상태를 적용하고, 현재 화면에는 `aria-current="page"`를 표시했습니다.
- **구현 결과**: 알림·운영 도구·외부 화면·새로고침 같은 현재 화면 기능 버튼은 기존 도구 영역에 유지했습니다.
- **정렬 보완**: 기능 버튼을 데스크톱 헤더 우측으로 이동하고, 탭 영역을 전체 폭으로 확장해 관리자·후기 탭 끝선과 색상 헤더 끝선을 맞췄습니다. 탭과 헤더 사이 하단 회색선을 제거했습니다.
- **검증 결과**: 세 화면의 데스크톱 및 390px 모바일 뷰포트에서 최상단 탭, 숨김 화면 제목, 기능 버튼 분리, 문서 가로 넘침 없음, 콘솔 오류 없음을 확인했습니다.
- **캐시 갱신**: 최상단 탭과 제목 숨김 CSS가 운영 환경에 반영되도록 `service-worker.js` 캐시 버전을 `kiosk-cache-v137`으로 올렸습니다.
- **운영 수동 검증 완료**: 실제 관리자 화면에서 세 탭 이동, 새로고침 후 활성 탭, 모바일 탭 터치, 기존 기능 메뉴 동작을 확인했습니다.

### P3 - 주방 간식 재고 현황 복구 및 그룹 표시 (구현 및 수동 검증 완료)

- `getSnacks` 조회와 성공 렌더링을 연결해 주방 재고 영역에 실제 데이터를 표시합니다.
- `target=user`는 일반 키오스크, `target=guest`는 배달왔삼 그룹으로 표시합니다.
- 각 항목에는 간식명과 재고만 표시하고, 재고가 0 이하이면 연한 빨간색 하이라이트를 적용합니다.
- 기존 15초 `getSnacks` 캐시를 활용하고, 최종 `placeOrder()`의 서버 재고 검증은 유지합니다.
- 재고 조회를 주문·이용자 조회와 분리해 한쪽 실패가 다른 영역을 덮어쓰지 않도록 했습니다.
- **수동 검증 완료**: 일반 키오스크·배달왔삼 그룹 분류, 간식명·재고 표시, 재고 0 하이라이트, 모바일 1열 표시를 확인했습니다.

### P4 - 외부 화면 빠른 버튼 색상 실험 (구현 완료, 수동 검증 대기)

- 관리자·주방·후기 화면 모두 일반 키오스크·배달왔삼·전광판 이동 링크를 기능 버튼과 분리해 헤더 좌측에 배치했습니다.
- 일반 키오스크 주황색, 배달왔삼 청록색, 전광판 보라색 링크를 사용하고 모바일 3열 배치를 적용했습니다.
- 주방에서는 사용 빈도가 높은 빌지 인쇄를 운영 도구 메뉴에서 우측 기능 버튼으로 분리했습니다.
- 관리자·후기 화면에는 빌지 인쇄를 노출하지 않고 화면별 기능 버튼만 유지합니다.
- **남은 작업**: 세 화면의 외부 링크 이동과 모바일 배치를 다시 수동 검증합니다.

### P5 - 주방 재고 게이지 표시 (구현 및 수동 검증 완료)

- **표시 방식**: 현재 흰색 재고 카드는 유지하고, 내부에 단색 채움 영역을 추가합니다. 간식명과 재고 숫자는 채움 영역 위에 표시합니다.
- **채움 계산**: `fillPercent = Math.min(Math.max(stock, 0) / 10, 1) * 100`으로 계산합니다. 10개 이상은 항상 100%입니다.
- **상태 색상**: 재고 10개 이상은 연초록(`#DEF7EC`), 4~9개는 연노랑/연주황(`#FEF3C7`), 1~3개는 연빨강(`#FEE2E2`)으로 표시합니다. 재고 0은 채움 없이 기존의 붉은 테두리·숫자(`is-zero` 클래스)로 표시합니다.
- **정보 역할**: 게이지는 부족 여부를 빠르게 보여주는 상태 표현이고, 정확한 수량은 숫자로 확인합니다. 10개와 20개의 게이지가 같아지는 한계는 숫자 표시로 보완합니다.
- **데이터 범위**: `getSnacks` 응답, 게스트·일반 그룹, 15초 캐시, 최종 `placeOrder()` 재고 방어는 변경하지 않습니다.
- **반응형 기준**: 데스크톱·모바일 모두 간식명과 숫자가 채움 색상에 묻히지 않아야 하며, 긴 간식명은 말줄임 처리합니다.
- **수동 검증**: 0개, 1~3개, 4~9개, 10개, 10개 초과 항목의 게이지 및 색상 렌더링 상태를 모두 로컬 서버를 통해 정상 검증 및 화면을 캡처했습니다.

### 완료 - 관리자 화면 간식 관리 테이블 슬림 재고 게이지 적용

- **개선 사유**: 간식 관리 테이블 내에 존재하던 중복 정보 태그(`[일반/게스트]`, `[판매중/숨김]`)를 제거하여 불필요한 시각적 노이즈를 배제했습니다.
- **표시 방식**: 확보된 중간 영역에 얇은 가로 프로그래스 바(Gauge)를 추가하고, 수량에 따른 상태 색상(10개 이상 초록 `#38A169`, 4~9개 황색 `#D69E2E`, 1~3개 적색 `#E53E3E`, 0개 미적용)을 적용했습니다.
- **구조 조정**: 테이블 헤더 및 행 구조를 6열에서 5열로 일체 조정하여 레이아웃을 간소화하고 가독성을 극대화했습니다.

### 완료 - 관리자 화면 내 이용자-간식 관리 테이블 레이아웃 UI 일원화 및 5열 단일화

- **개선 사유**: 이용자 테이블과 간식 테이블의 수치 조작 인터페이스(증감 조정 버튼, 수동 입력창, 수정 액션 버튼)가 서로 달라 발생하던 시각적 복잡도를 제거하고, 버튼들이 두 줄로 쪼개져 밀리던 레이아웃 문제를 해결했습니다.
- **조정 내역**: 
  - **이용자 테이블 5열 개편**: 중복 노출되던 `상태` 열(활성/비활성 배지)을 삭제하고 **가로 5개 열** 구조로 슬림화했습니다.
  - **간식 테이블 5열 개편**: 기존에 나뉘어 있던 `재고 수준`(게이지) 열과 `현재 재고`(수량 텍스트) 열을 하나로 병합하여 **가로 5개 열** 구조로 개편하고, 헤더 타이틀을 2글자 **`재고`**로 단축하여 불필요한 열 이름 줄바꿈 현상을 방지했습니다.
  - **1열 강제 정렬 적용**: 조작 버튼부 및 빠른 증감 영역에 가로 1열 고정 스타일(`.admin-flex-nowrap`, `white-space: nowrap`)을 입히고, 제어 버튼의 라벨을 축소(`정보 수정` -> `수정`, `숨기기` -> `숨김`, `판매중` -> `판매`)하여 강제 한 줄 배치가 보장되도록 수정했습니다.
- **통합 일관성**: 두 관리 테이블이 물리적으로 완벽한 5열 구조 대칭을 이룸으로써 대시보드의 균형미와 가독성, 사용성이 극대화되었습니다.

### 완료 - 이용자 관리 탭 검색란 제거 및 상단 컨트롤러 UI 대칭 매칭

- **개선 사유**: 간식 관리 탭에는 검색 기능이 존재하지 않아 두 탭의 상단 디자인 구조에 불균형이 있었습니다. 또한 매점 운영 특성상 이용자 검색 기능의 효용성이 낮아 이를 제거하여 디자인 대칭을 맞췄습니다.
- **조정 내역**: 
  - 이용자 탭 내 `search-user-input` 검색 상자를 완전히 제거했습니다.
  - 타이틀(`🪙 이용자 크레딧 관리`) 옆으로 `비활성 이용자 표시` 체크박스와 `➕ 신규 이용자 등록` 버튼이 나란히 정렬되도록 레이아웃을 전면 개편했습니다.
  - 이로써 두 탭의 상단 좌우 대칭 구조가 완전히 일치하게 정돈되었습니다.

### 완료 - 후기 관리 화면 카드 뷰 레이아웃 개편 및 주문번호 은닉

- **개선 사유**: 기존의 가로형 표(Table) 형식이 모바일/태블릿 등 다양한 가로 폭 기기에서 깨지는 문제를 예방하고 긴 후기 글 및 이미지 썸네일을 시각적으로 가독성 있게 표현하기 위해 개편을 단행했습니다.
- **조정 내역**: 
  - **반응형 카드 뷰 적용**: 2열 그리드(화면이 좁을 때는 1열) 형태의 부드러운 화이트 톤 카드 상자로 전면 교체했습니다.

  - **정보 병합 및 답글 세련화**: 작성일, 게스트명, 칭찬 스탬프 및 공개 배지들을 깔끔한 칩 형태로 카드 상단에 배치하고, 하단에 연노랑 답글 상자를 정렬해 관리 편의성을 향상했습니다.
