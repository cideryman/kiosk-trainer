## 상담 후 다음 모바일 개선 큐

### M-P1 - 관리자 로그인 게이트 검토·구현 (구현 및 배포 후 검증 완료)

- 관리자·주방·후기 화면에서 인증 전 본문과 데이터를 숨기고, 인증 성공 후에만 화면을 표시하는 전체 화면 게이트를 적용했습니다.
- 기존 `AdminAuth`를 재사용하고, 세 화면의 초기 데이터 조회를 인증 성공 이후로 이동했습니다.
- 관리자 이미지를 로그인 게이트와 `admin-180/192/512.png` 공통 PWA 아이콘에 적용했습니다.
- 같은 브라우저 탭·PWA 세션에서는 인증 상태를 유지하고, 잠그기 또는 권한 거절 시 게이트로 돌아갑니다.
- 인증 성공 후 기존 인증 바는 숨기고, PC에서는 기능 버튼의 마지막에 열쇠 버튼을 두며 모바일에서는 세 열 그리드의 마지막 칸에 고정했습니다.
- **배포 후 검증 완료**: 세 URL 직접 접속 시 정보 비표시, 올바른 키·잘못된 키, 탭 이동 후 재인증 여부, PC·모바일 열쇠 위치와 잠그기, 이미지 크기와 PWA 아이콘을 확인했습니다.

### M-P2 - 게스트 주문 운영 패널 모바일 재배치 (구현 및 수동 검증 완료)

- 상태 정보와 기본 설정을 모바일 2열 그룹으로 묶고, 기본 배달지는 전체 폭으로 배치했습니다.
- 배달팀 설정은 제목·저장, 표시 여부·타이틀, 담당자 입력, 한마디 입력 영역으로 나누고, 운영 시간 버튼은 모바일 2x2 그리드로 정리했습니다.
- 모바일 기본 설정 입력 폭과 라벨 줄바꿈을 조정해 `배달비`가 기본 크레딧 영역을 침범하지 않도록 보완했고, 주방 기능 버튼 문구를 `📊 운영 결과`로 줄였습니다.
- 게스트 운영 설정의 기본 크레딧 라벨을 `기본 🪙`으로 줄여 모바일 표시 폭을 추가로 확보했습니다.
- 배달팀 설정 문구를 `표시`·`제목:`으로 줄이고, 직접 입력 제어를 `⏱ 직접 열기:` 한 줄로 정리했습니다.
- 모바일 `표시`·`제목:` 행을 flex 배치로 바꾸고 제목 입력 폭을 확장했으며, 메시지 입력 글자 크기도 조정했습니다.
- 운영 상태·마감 예정·남은 시간과 20/30/60분 열기·즉시 마감은 항상 표시하고, 상세 설정은 기본 접힘 `⚙ 운영 설정` 패널로 분리했습니다.
- PC에서는 기존 가로 배치를 유지하고 모바일에서만 패널 여백·줄바꿈·입력 폭을 조정했습니다.
- 자동 새로고침 중 입력 중인 설정값을 덮어쓰지 않는 동작을 유지합니다.
- **수동 검증 완료**: 모바일 390px·320px에서 입력 영역, 저장, 20/30/60분 열기, 마감, 직접 입력 열기와 PC 레이아웃 회귀를 확인했습니다.

### M-P3 - 주방 상단 요약 카드 문구·값 분리 (구현 및 수동 검증 완료)

- `오늘 누적 크레딧`·`오늘 총 주문`으로 제목을 줄이고, 값은 `🪙 숫자`·`숫자건` 형식으로 다음 줄에 표시합니다.
- 현재 2열 카드 구조와 통계 의미는 유지합니다.
- 모바일에서 제목은 한 줄로 유지하고 카드 안쪽 여백과 값 글자 크기를 조정했습니다.
- **수동 검증 완료**: PC·390px·320px에서 제목 줄바꿈 없음, 실제 누적 크레딧·주문 수 갱신, 탭 전환 후 값 유지를 확인했습니다.

### M-P4 - 주방 화면 주문·통계·재고 탭 분리 (구현 및 수동 검증 완료)

- 주문 탭에는 간식 챙겨주기와 제공 완료 목록을 배치하고, 통계 탭에는 간식별 집계·이용자별 주문 건수를 배치했습니다.
- 재고 탭에는 현재 간식 재고 현황을 배치했습니다.
- PC·모바일 모두 같은 3탭 구조를 사용하며, 오늘 요약 카드·게스트 운영 설정·새로고침 제어는 탭 밖에 유지합니다.
- 기존 조회·자동 새로고침·주문 처리 ID와 렌더링 로직은 유지하고 탭 표시만 전환합니다.
- **수동 검증 완료**: PC·390px·320px에서 세 탭 전환, 주문 처리·필터, 통계·재고 표시, 자동 새로고침 후 탭 유지, 가로 넘침 없음을 확인했습니다.

### 완료 - 주문보관 행 수 경고 및 안전 실행 확인

- `getOrdersToday` 응답에 실제 `주문내역` 데이터 행 수를 포함해 주방 화면에서 운영 시트 규모를 확인할 수 있게 했습니다.
- 100행 이상이면 `주문보관 권장`, 300행 이상이면 `주문보관 필요` 경고를 표시합니다. 경고는 오늘 주문 수가 아니라 `주문내역` 전체 데이터 행 수를 기준으로 합니다.
- `보관 점검`은 시트를 변경하지 않으며, 점검 기준을 통과한 경우에만 `지난 주문 보관` 버튼을 활성화합니다.
- 보관 실행 확인 문구는 영어 문자열 대신 `주문보관확인`을 사용합니다.
- 기존 보관 시트는 실행 직전에 자동 백업하고, 중복 제거·23열 병합·주문내역 정리 로직을 거친 뒤 보관합니다.
- **배포 후 검증 완료**: 99행 이하 숨김, 100행 이상 권장, 300행 이상 필요 표시와 한글 확인 문구를 확인했습니다.

### P13 - 배달왔삼 이용 신청·승인 관리 (디자인 고도화 완료, GAS 배포·운영 검증 대기)

- **목적**: 기존 복지관 봉사자·후원자와 관리자가 인정하는 관계자가 배달왔삼 이용 가능 여부를 사전에 신청하고, 관리자가 배달 장소를 확인한 뒤 수동으로 안내합니다. 신청은 메뉴 주문이 아니며 기존 주문 화면의 접근을 기술적으로 제한하지 않습니다.
- **공개 화면 및 디자인**: `guest-apply.html`에 소개, 활동 의미, 이용 절차, 운영 안내, 신청서를 스토리형 화면으로 구성했습니다. 신규 일러스트 3종(`meaning-1.jpg`, `meaning-2.jpg`, `meaning-3.jpg`)을 각 의미 카드 상단에 둥글고 경계선 없는 Full-bleed 형태로 연동했으며, 헤더 브랜드명을 듀얼 컬러("배달왔"은 `--teal`, "삼"은 `--orange`)로 칠해 브랜딩을 강화했습니다.
- **정렬 및 반응형**: Hero 콘텐츠와 본문 섹션의 정렬 축을 수평 너비 `1120px`로 통일해 PC 뷰의 정렬 정합성을 개선했습니다. 모바일 뷰에서 곰돌이 마스코트 캐릭터(**달곰이**)의 머리가 잘려 보이는 현상을 방지하기 위해 `object-position: center top`과 이미지 높이 상향(`155px`)을 적용했습니다.
- **신청 항목**: 이름·관계·연락처·배달 장소·희망 요일·개인정보 동의는 필수이고, 관계 설명·상세 전달 방법·전달 메시지는 선택입니다. 메뉴·수량·음료 선호는 받지 않습니다.
- **브라우저 저장 원칙**: 신청 개인정보는 `localStorage`·`sessionStorage`에 저장하지 않습니다. 응답 유실 재시도에 필요한 `requestId`만 현재 탭의 `sessionStorage`에 보관하고 접수 성공 후 삭제합니다.
- **데이터 계약**: 신규 `이용신청` 시트는 19열이며 `createdAt, applicationId, requestId, name, relationType, relationDetail, phone, deliveryPlace, deliveryDetail, preferredDays, message, consentAt, status, contactedAt, reviewedAt, retentionUntil, anonymizedAt, adminMemo, updatedAt` 순서를 사용합니다.
- **GAS 모듈**: `gas/12_GuestApplications.gs`가 시트 준비, 공개 설정, 신청 접수, 멱등성, 동일 연락처 중복 차단, 관리자 조회·처리, 운영 설정, 만료 점검·익명화를 담당합니다. 공개 요청 본문과 개인정보는 GAS 로그 및 관리자로그에 남기지 않습니다.
- **신청번호와 중복 방지**: `LockService` 안에서 `APP-YYYYMMDD-순번`을 만들고, 같은 `requestId` 재전송에는 기존 신청번호를 반환합니다. 익명화되지 않은 동일 연락처 신청은 새 행을 만들지 않고 일반적인 중복 안내만 반환합니다.
- **상태 흐름**: `PENDING(대기)`, `APPROVED(승인)`, `REJECTED(반려)`, `INACTIVE(중지)`만 사용합니다. 반려·중지 시 처리일부터 30일 뒤 `retentionUntil`을 기록하고, 재승인하면 만료 예정일을 해제합니다.
- **관리자 화면**: `admin.html` 탭을 `이용자 / 간식 / 순서 / 신청`으로 구성했습니다. 신청 탭은 대기 우선 정렬, 상태 필터·건수, 마스킹 목록, 전체 개인정보 상세 모달, 승인·반려·중지·연락 완료·메모 저장, 신청 안내 설정을 제공합니다.
- **익명화**: 만료 점검 결과를 확인한 뒤 `신청정보정리`를 입력해야 실행됩니다. 익명화 시 신청번호·상태·처리 시각은 남기고 이름·연락처·장소·관계·희망 요일·메시지·메모·`requestId`를 제거합니다.
- **운영 연결**: `guest.html` 하단에 낮은 강조도의 이용 신청 링크를 추가했고, 신청 페이지는 게스트 Manifest와 아이콘을 재사용합니다. 별도 PWA나 승인 코드, 신청자 상태 조회, 자동 문자·카카오 발송은 도입하지 않았습니다.
- **배포 순서**:
  1. 운영 DB 백업 후 GAS에 `00_Config.gs`, `01_Router.gs`, 신규 `12_GuestApplications.gs`, `90_Diagnostics.gs`를 반영합니다.
  2. GAS 편집기에서 `setupGuestApplicationSheet()`를 한 번 실행하고 `이용신청` 19열 헤더를 점검합니다.
  3. GAS 새 버전을 배포한 뒤 신청 설정은 기본값인 `마감`으로 둡니다.
  4. 관리자 목록·상세·설정과 공개 마감 화면을 검증한 뒤 정적 파일을 GitHub에 반영합니다.
  5. 개인정보 안내 문구를 기관 담당자가 최종 확인한 뒤 관리자 신청 탭에서 `이용 신청 받기`를 켭니다.
- **자동 검증 완료**: GAS 결합 구문, HTML 인라인 스크립트, PC·390px·320px 가로 넘침, 공개 Mock 접수 성공, 관리자 목록·상세 모달, 최초 접수·멱등 재전송·동일 연락처 차단·마스킹·30일 만료일 생성을 확인했습니다. 또한 신규 일러스트 3종, 헤더 포인트 채색, PC/모바일 정렬 개선 및 캐릭터 명칭 확정(**달곰이**)을 마쳤습니다.
- **운영 수동검증 필요**: 실제 Sheets에서 19열 생성, 공개 신청 1건 접수, 관리자 상태 변경·연락 완료·설정 저장, 만료일과 익명화, 비인증 API 차단, 기존 로컬·카카오 주문 회귀를 확인합니다.

### P9 - 배달왔삼 선택형 TTS 안내 검토 (조건부 권장, 구현 보류)

- **목적**: 배달왔삼 이용자의 접근성을 높이되, 반복 음성이나 개인정보 노출로 운영을 방해하지 않는 범위에서 음성 안내를 검토합니다.
- **적용 조건**: 음성 안내는 선택 기능으로 제공하고 기본값은 끕니다. 시작·주문 완료·주문 상태 변경·취소처럼 중요한 순간에만 제한적으로 안내합니다.
- **제외 범위**: 간식 클릭마다 음성 재생, 폴링 때마다 반복 안내, 주문 상세정보의 상시 방송은 적용하지 않습니다.
- **기술 판단**: `js/app.js`의 기존 `AppState.speak()`와 `ttsEnabled` 설정을 재사용할 수 있어 백엔드·DB 변경 없이 진행할 수 있습니다.
- **주의사항**: 브라우저 자동 재생 제한, 기기별 한국어 음성 차이, 주변에 닉네임·주문번호가 들릴 수 있는 개인정보 노출 가능성을 먼저 확인합니다.
- **착수 조건**: 실제 이용자에게 필요한 안내 문구를 정한 뒤, 배달왔삼 시작 화면에서 토글과 핵심 안내만 단계적으로 구현합니다.

### P10 - 관리자 인증 종료 버튼 의미 개선 (구현 완료, 수동 검증 대기)

- 기존 `🔑`는 관리자 키 입력·인증 기능으로도 해석될 수 있어 동작이 모호했습니다.
- 인증이 완료된 세 관리자 화면의 기능 행에는 관리자 세션을 끝내고 관문으로 나가는 `🚪` 버튼을 표시합니다.
- `🚪`를 누르면 인증이 종료되고 화면이 게이트로 전환됩니다. 게이트의 `🔒`는 잠긴 상태를 보여주는 표시로만 사용합니다.
- 버튼의 툴팁과 접근성 이름은 `관리자 나가기`로 통일해 동작을 명확히 합니다.
- **수동 검증 필요**: 관리자·주방·후기 화면에서 잠금 아이콘, 툴팁, 클릭 후 게이트 전환, 모바일 버튼 위치를 확인합니다.

### P5 - 주방 재고 게이지 표시 (구현 및 수동 검증 완료)

- **표시 방식**: 현재 흰색 재고 카드는 유지하고, 내부에 단색 채움 영역을 추가합니다. 간식명과 재고 숫자는 채움 영역 위에 표시합니다.
- **채움 계산**: `fillPercent = Math.min(Math.max(stock, 0) / 10, 1) * 100`으로 계산합니다. 10개 이상은 항상 100%입니다.
- **상태 색상**: 재고 10개 이상은 연초록(`#DEF7EC`), 4~9개는 연노랑/연주황(`#FEF3C7`), 1~3개는 연빨강(`#FEE2E2`)으로 표시합니다. 재고 0은 채움 없이 기존의 붉은 테두리·숫자(`is-zero` 클래스)로 표시합니다.
- **정보 역할**: 게이지는 부족 여부를 빠르게 보여주는 상태 표현이고, 정확한 수량은 숫자로 확인합니다. 10개와 20개의 게이지가 같아지는 한계는 숫자 표시로 보완합니다.
- **데이터 범위**: `getSnacks` 응답, 게스트·일반 그룹, 15초 캐시, 최종 `placeOrder()` 재고 방어는 변경하지 않습니다.
- **반응형 기준**: 데스크톱·모바일 모두 간식명과 숫자가 채움 색상에 묻히지 않아야 하며, 긴 간식명은 말줄임 처리합니다.
- **수동 검증**: 0개, 1~3개, 4~9개, 10개, 10개 초과 항목의 게이지 및 색상 렌더링 상태를 모두 로컬 서버를 통해 정상 검증 및 화면을 캡처했습니다.

### 완료 - 관리자 화면 간식 관리 테이블 슬림 재고 게이지 적용

- **개선 사유**: 간식 관리 테이블 내에 존재하던 중복 정보 태그(`[일반/게스트]`, `[판매중/숨김]`)를 제거하여 불필요한 시각적 노이즈를 배제했습니다.
- **표시 방식**: 확보된 중간 영역에 얇은 가로 프로그래스 바(Gauge)를 추가하고, 수량에 따른 상태 색상(10개 이상 초록 `#38A169`, 4~9개 황색 `#D69E2E`, 1~3개 적색 `#E53E3E`, 0개 미적용)을 적용했습니다.
- **구조 조정**: 테이블 헤더 및 행 구조를 6열에서 5열로 일체 조정하여 레이아웃을 간소화하고 가독성을 극대화했습니다.

### 완료 - 관리자 화면 내 이용자-간식 관리 테이블 레이아웃 UI 일원화 및 5열 단일화

- **개선 사유**: 이용자 테이블과 간식 테이블의 수치 조작 인터페이스(증감 조정 버튼, 수동 입력창, 수정 액션 버튼)가 서로 달라 발생하던 시각적 복잡도를 제거하고, 버튼들이 두 줄로 쪼개져 밀리던 레이아웃 문제를 해결했습니다.
- **조정 내역**: 
  - **이용자 테이블 5열 개편**: 중복 노출되던 `상태` 열(활성/비활성 배지)을 삭제하고 **가로 5개 열** 구조로 슬림화했습니다.
  - **간식 테이블 5열 개편**: 기존에 나뉘어 있던 `재고 수준`(게이지) 열과 `현재 재고`(수량 텍스트) 열을 하나로 병합하여 **가로 5개 열** 구조로 개편하고, 헤더 타이틀을 2글자 **`재고`**로 단축하여 불필요한 열 이름 줄바꿈 현상을 방지했습니다.
  - **1열 강제 정렬 적용**: 조작 버튼부 및 빠른 증감 영역에 가로 1열 고정 스타일(`.admin-flex-nowrap`, `white-space: nowrap`)을 입히고, 제어 버튼의 라벨을 축소(`정보 수정` -> `수정`, `숨기기` -> `숨김`, `판매중` -> `판매`)하여 강제 한 줄 배치가 보장되도록 수정했습니다.
- **통합 일관성**: 두 관리 테이블이 물리적으로 완벽한 5열 구조 대칭을 이룸으로써 대시보드의 균형미와 가독성, 사용성이 극대화되었습니다.

### 완료 - 이용자 관리 탭 검색란 제거 및 상단 컨트롤러 UI 대칭 매칭

- **개선 사유**: 간식 관리 탭에는 검색 기능이 존재하지 않아 두 탭의 상단 디자인 구조에 불균형이 있었습니다. 또한 매점 운영 특성상 이용자 검색 기능의 효용성이 낮아 이를 제거하여 디자인 대칭을 맞췄습니다.
- **조정 내역**: 
  - 이용자 탭 내 `search-user-input` 검색 상자를 완전히 제거했습니다.
  - 타이틀(`🪙 이용자 크레딧 관리`) 옆으로 `비활성 이용자 표시` 체크박스와 `➕ 신규 이용자 등록` 버튼이 나란히 정렬되도록 레이아웃을 전면 개편했습니다.
  - 이로써 두 탭의 상단 좌우 대칭 구조가 완전히 일치하게 정돈되었습니다.

### 완료 - 후기 관리 화면 카드 뷰 레이아웃 개편 및 주문번호 은닉
