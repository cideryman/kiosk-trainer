# 안정된 결정과 변경 금지선

[핸드오프로 돌아가기](../../handoff.md)

## 데이터와 개인정보

- 간식 `제공대상`은 `user` 또는 `guest`만 사용하고 `both`는 사용하지 않습니다.
- 게스트 주문의 `userId`는 항상 `guest`로 저장합니다. 카카오 식별값으로 대체하지 않습니다.
- 카카오 원본 ID와 토큰은 Sheets에 저장하지 않고 솔트를 사용한 단방향 값 `guestKey`만 저장합니다.
- 실명, 이메일, 전화번호는 수집하지 않습니다.
- 브라우저 저장소는 주문 편의, 중복 주문 방지, 주문 조회, 게스트 크레딧 중복 사용 방지에 필요한 값만 사용합니다.
- 로컬 게스트 표시명은 기기 안에 저장할 수 있지만 배달지, 실명, 연락처, 원본 카카오 ID, 크레딧은 새로 로컬 저장하지 않습니다.

## Sheets 구조 보호

- 실제 운영 DB 열을 코드 가정에 맞춰 임의로 이동하거나 삭제하지 않습니다.
- 운영 점검 경고만 보고 열을 재배열하지 않습니다. alias 호환과 최신 DB 헤더를 먼저 확인합니다.
- 시트 구조 보정이나 마이그레이션을 `onOpen` 또는 시간 기반 트리거에 연결하지 않습니다.
- 구조 변경은 백업 확보, 레이아웃 대조, 운영 중지 또는 낮은 운영 시간 확보 후 수동으로 실행합니다.
- 주문 보관은 주 1회 또는 `주문내역` 약 50행 이상을 권장하며 운영이 없는 시간에 주방 `운영 도구 > 지난 주문 보관`으로 수동 실행합니다.

## 주문 안정성

- 프론트 주문 버튼 잠금은 UX 방어선이며 서버 중복 방지의 대체물이 아닙니다.
- `placeOrder`는 `idempotencyKey`로 동일 요청 재전송을 식별하고 성공한 기존 주문 결과를 돌려줍니다.
- 정상적인 새 주문은 새 키를 만들기 때문에 차단하지 않습니다.
- 주문 성공 여부가 불명확한 네트워크 오류에서는 같은 키를 유지하고, 명확한 성공·실패 뒤에는 키를 폐기합니다.
- 최종 주문은 표시 캐시가 아니라 원본 Sheets의 재고, 판매상태, 크레딧을 다시 검증합니다.
- 트랜잭션 롤백과 부분 실패 복구는 멱등성과 분리합니다. 실제 불일치 증거 없이 대규모 구조를 도입하지 않습니다.

## 배달왔삼 UX와 운영

- 주문 시작 단계에서 `guestDeviceId` 또는 카카오 `guestKey` 기반 크레딧 조회를 유지합니다. 이는 새 화면이 아니라 백그라운드 조회입니다.
- 로컬 게스트 닉네임 입력은 주문 시작이 아니라 마지막 `주문자 정보` 영역에서 받습니다.
- 카카오 저장 프로필이 있으면 저장된 표시명을 사용하고 입력칸을 생략합니다.
- 영업 종료 중 메뉴 보기는 `menu.html?browse=guest` 읽기 전용 화면으로 제공하며 주문 세션, 크레딧, 장바구니를 만들지 않습니다.
- 예약 종료 전에 시작한 주문은 종료 후 5분까지 제출할 수 있습니다. 관리자의 즉시 마감은 진행 중 주문도 거절합니다.
- 마감 상태 API를 확인할 수 없으면 읽기 전용 메뉴 버튼을 노출하지 않습니다.

## 성능과 캐시

- 재고·크레딧·주문 원본 데이터를 긴 TTL로 캐시하지 않습니다.
- `getSnacks` 15초 캐시는 메뉴 표시용이며 변경 경로에서 즉시 무효화합니다.
- 주문 읽기 2초 캐시는 폴링 부하 완화용이며 주문 상태 변경 때 무효화합니다.
- 설정 캐시는 변경 API와 무효화가 항상 짝을 이뤄야 합니다.
- 성능만을 이유로 `placeOrder`의 락 범위나 처리 순서를 크게 바꾸지 않습니다.

## 화면과 관리 책임

- `admin.html`: 이용자, 간식, 크레딧, 간식 표시 순서
- `kitchen.html`: 실시간 주문, 당일 통계, 인쇄, 주문 보관, 게스트 영업과 배달팀
- `reviews.html`: 후기 확인, 공개·숨김, 관리자 대리 답글
- 관리자·주방·후기 화면의 상단 하위 메뉴는 한 번에 하나만 열며 다른 상단 버튼, 바깥 클릭, 메뉴 선택, `Esc`에서 닫힙니다.
- 후기 답글은 관리자 후기 화면에서 교사가 의견을 취합해 대리 작성합니다. 주방 화면에서 훈련생이 직접 실시간 작성하는 흐름은 만들지 않습니다.

## 미디어와 정적 자원

- 관리자 신규 이미지 업로드는 브라우저에서 최대 800px, WebP 품질 0.82로 변환합니다. 애니메이션 GIF는 원본을 유지합니다.
- 웹폰트 CDN을 사용하지 않습니다. 기기에 Pretendard가 있으면 사용하고 없으면 시스템 한글 폰트로 대체합니다.
- 정적 파일을 바꾸면 서비스 워커 캐시 버전을 검토합니다.
