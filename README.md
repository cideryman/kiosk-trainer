# 🍪 주간보호센터 발달장애인용 간식 키오스크 (PWA 지원)

이 프로젝트는 주간보호센터의 발달장애인 이용자들이 직관적이고 친근한 방식으로 원하는 간식을 고르고, 크레딧(동전)을 사용해 주문할 수 있는 키오스크 시스템입니다. 

---

## 🌟 주요 특징

### 1. 발달장애인 맞춤형 UI/UX
- **크고 선명한 컴포넌트**: 모든 버튼의 최소 높이를 `48px` 이상(`72px` 권장)으로 구성하고 폰트 크기를 키웠습니다.
- **햅틱 촉각 피드백**: 터치 스크린 상에서 버튼을 탭하거나, 재고/크레딧 부족 시 고유한 스마트폰 진동을 통해 성공/실패 여부를 몸으로 직관할 수 있게 하였습니다.
- **시각 보완 시스템**: 이용자들의 얼굴 사진 또는 직관적인 이름 첫 글자(원형 파스텔 이니셜 아바타)를 함께 매칭하여 문해력에 상관없이 자기를 바로 찾을 수 있도록 돕습니다.
- **대체 이모지**: 간식 이미지가 없을 때도 간식의 이름에 맞는 대형 푸드 이모지(🍪, 🧃 등)가 자동으로 매핑되어 제공됩니다.

### 2. PWA (Progressive Web App) 공식 지원
- **홈 화면에 추가**: 모바일 브라우저(Android Chrome, iOS Safari)로 접속 후 주소창 메뉴에서 **"홈 화면에 추가"** 또는 **"앱 설치"**를 눌러 태블릿이나 스마트폰 화면에 진짜 앱처럼 설치할 수 있습니다.
- **오프라인 동작 (정적 자원 캐싱)**: 서비스 워커(`service-worker.js`)를 통해 앱 구동 파일들(HTML, CSS, JS)을 로컬 기기에 항상 보관하므로 로딩 속도가 비약적으로 빠르며 네트워크 연결이 해제된 정적 상태에서도 첫 화면이 부드럽게 로딩됩니다.
- **독립 실행 (Standalone)**: 웹 브라우저의 주소창 및 하단 네비게이션바가 모두 가려지고 기기의 전체 화면으로 키오스크 모드가 실행됩니다.

### 3. 실시간 DB & 재고 제어
- **구글 스프레드시트 연동**: 재고 상태, 금액, 이미지 URL, 이용자별 크레딧이 DB(스프레드시트)에서 수정되는 즉시 API를 통해 실시간 반영됩니다.
- **품절 및 재고 한도 가드**: 재고가 없는 간식은 자동으로 흐려지고 `+` 버튼이 활성화되지 않으며, 장바구니에 담은 수량이 재고 수량을 넘어설 수 없습니다.
- **실패 예외 한국어 알림**: 크레딧(보유 금액) 부족 및 재고 한도 초과 오류 발생 시 화면에 크고 알기 쉽게 한국어로 명시적 안내를 띄웁니다.

## 📁 프로젝트 파일 구조

```
├── index.html            # 이용자 선택 화면
├── menu.html             # 간식 선택 화면 (재고 및 크레딧 제한)
├── confirm.html          # 주문 확인 화면 (에러 핸들링)
├── complete.html         # 주문 완료 화면 (타이머 리디렉션)
├── admin.html            # 관리자 오늘 주문 대시보드 & 실시간 간식 재고
├── manifest.json         # PWA 웹앱 설정 마니페스트
├── service-worker.js     # PWA 오프라인 캐싱용 서비스 워커
├── css/
│   └── style.css         # 발달장애인 접근성 특화 가독성 CSS
├── icons/
│   ├── icon-192.png      # 192px PWA 앱 아이콘
│   └── icon-512.png      # 512px PWA 앱 아이콘
└── js/
    ├── config.js         # API 주소 설정 & USE_MOCK 모드 스위치
    └── app.js            # 공통 상태 관리, 진동 피드백 및 서비스 워커 등록
```

## 🛠️ 연동 DB 구조 명세

앱을 안정적으로 서비스하기 위해서는 구글 스프레드시트 또는 API DB 테이블이 아래의 스키마 명세를 따라야 합니다:

### 1. 이용자목록 (Users)
- **이용자ID** (`userId`): 고유식별코드
- **별명** (`nickname`): 화면에 표시될 크고 쉬운 이름
- **크레딧** (`credit`): 남은 사용 가능한 동전 수치
- **사용여부** (`useYn`): 'Y' 또는 'N'
- **사진URL** (`imageUrl`): 이용자의 정면 사진 URL (공백 가능)

### 2. 간식목록 (Snacks)
- **간식ID** (`snackId`): 고유식별코드
- **이름** (`name`): 간식의 한글 이름 (예: 초코칩 쿠키)
- **포인트** (`point`): 차감할 크레딧 수치
- **사진URL** (`imageUrl`): 간식 제품 사진 URL (공백 가능)
- **판매여부** (`saleYn`): 'Y' 또는 'N'
- **재고** (`stock`): 정수형 잔여 개수 (`0` 이면 품절)

### 3. 주문내역 (Orders)
- **주문시간** (`timestamp`): ISO 8601 포맷 타임스탬프
- **주문번호** (`orderId`): 주문 생성 코드
- **이용자ID** (`userId`)
- **별명** (`nickname`)
- **간식ID** (`snackId`)
- **간식명** (`snackName`)
- **수량** (`quantity`)
- **차감포인트** (`point`)

## 🚀 실행 및 테스트 방법

1. **정적 서빙 실행**:
   - 이 프로젝트는 순수 정적 파일들(Pure HTML/JS/CSS)로 빌드되어 있습니다.
   - VS Code의 `Live Server` 플러그인 또는 Python 정적 서버(`python -m http.server 8080`) 등을 사용해 로컬 또는 호스팅 주소에서 가볍게 띄울 수 있습니다.
   
2. **API 연동 상태 제어**:
   - `js/config.js` 상단의 `const USE_MOCK = false;` 변수를 제어하여 테스트할 수 있습니다.
   - `false`일 때는 실제 구글 시트 API를 직접 찔러 통신 오류 여부 및 실제 데이터를 불러옵니다.
   - `true`로 설정하면 로컬 환경에서 재고 제한 및 품절, 크레딧 제어를 완벽하게 가상 시뮬레이션으로 테스트(Mocking)할 수 있습니다.

3. **PWA 등록 확인**:
   - 브라우저 개발자 도구(F12) -> Application 탭에서 `Manifest`가 올바르게 호출되었는지, `Service Workers`가 Active 상태로 돌고 있는지 체크하십시오.
   - PWA가 정상 세팅되면 모바일 혹은 PC 브라우저 주소창 오른쪽에 **[앱 설치]** 모양 아이콘이 자동 생성됩니다.
