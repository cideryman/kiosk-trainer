# 삼각지 카페 PWA 간식 키오스크 & 배달 서비스 시스템

주간보호센터 발달장애인 이용자가 스스로 간식을 주문하고 결제해보는 **키오스크 모드**와, 이용자가 직접 배달원이 되어 배달을 수행할 수 있는 **게스트/배달 모드(배달왔삼)**가 결합된 Progressive Web App (PWA) 시스템입니다.

본 시스템은 발달장애인의 디지털 기기 적응(키오스크 연습)과 직업 재활 훈련을 돕기 위해 접근성과 다감각 피드백을 최우선으로 설계하였으며, 구글 스프레드시트를 백엔드 DB로 삼아 실시간으로 운영할 수 있습니다.

---

## 🎯 훈련 및 설계 철학 (Training & Design Philosophy)

본 시스템은 발달장애인을 위한 일방적인 '편의 제공'이나 '완충된 보호 구역'에 그치지 않고, **현실 사회의 기술 환경에 능동적으로 적응하기 위한 실전형 '훈련 도구(Trainer)'**로 설계되었습니다.

* **현실적인 터치 감각 및 스크롤 환경**:
  - 터치 떨림이나 불완전한 드래그 오작동을 인위적으로 보정해주는 과도한 감도 필터링을 의도적으로 배제했습니다.
  - 실제 매장에 설치된 표준 정전식 키오스크들과 동일하게 표준 브라우저의 마우스/터치 클릭 이벤트를 따르게 함으로써, 스크롤 도중 발생하는 오터치나 튕김 현상에 이용자가 직접 부딪히고 적응하며 올바른 상호작용 감각을 체득하도록 훈련 효과를 극대화했습니다.
* **직업적 자립 및 배달원 역할 강화**:
  - 단순 놀이식 체험을 탈피하고 배달원으로서의 직업 의식과 자부심을 심어주기 위해, "체험" 위주의 단어 대신 사실적인 **"배달 서비스" 브랜딩**을 지향합니다.

## 🚀 주요 기능

### 1) 이용자 및 키오스크 화면 (`index.html`, `menu.html`, `confirm.html`, `complete.html`)
* **발달장애인 맞춤형 UI**: 버튼, 이름 카드, 간식 카드의 클릭 영역을 큼직하게 구성하여 지체 장애나 터치 오작동을 최소화했습니다.
* **직관적인 시각 요소**: 크레딧 잔액을 건조한 숫자 대신 동전 이미지/토큰 그래픽으로 표시하여 이해를 돕습니다.
* **이미지 기반 장바구니**: 간식 이미지를 터치해도 장바구니 수량이 증가하며, 주문 확인 화면에서 담은 간식의 사진과 대체 이모지를 큼직하게 띄워줍니다.
* **다감각 피드백**: 버튼 클릭, 항목 추가/제거 시 효과음 및 햅틱 진동 피드백을 제공합니다.
* **음성 안내 (TTS)**: 주문 전반의 선택 사항과 잔액 부족, 품절 상태 등을 한국어 음성으로 실시간 안내합니다.
* **유휴 시간 자동 로그아웃 (Idle Timeout)**: 70초간 터치 이벤트가 없으면 화면이 어두워지며 10초간 TTS 음성 경고와 카운트다운 비프음이 울린 후, 장바구니와 세션을 자동 리셋하고 초기 화면으로 돌아갑니다.

### 2) 배달왔삼 게스트 모드 (`guest.html`, `guest-orders.html`)
* **체험형 배달 서비스**: "직업 체험" 대신 실제 카페 배달원처럼 활동할 수 있도록 브랜드를 **"배달왔삼 배달 서비스"**로 지향합니다.
* **주문표시명 입력**: 게스트 주문 시 부르기 쉬운 주문표시명을 직접 적도록 안내합니다. 필요할 때만 `랜덤 이름 쓰기` 버튼으로 재밌는 이름(`형용사 + 삼각지 + 동물/사물`)을 넣을 수 있습니다.
* **선택형 카카오 연결**: 카카오에서 실명/이메일/전화번호/배송지를 가져오지 않고, 카카오 서비스 사용자 ID만 내부 `guestKey`로 변환해 저장합니다. 연결한 사용자는 다른 기기에서도 오늘 주문을 확인할 수 있으며, 사용자가 별도 체크한 경우에만 직접 입력한 주문표시명과 마지막 배송지를 다음 주문용으로 기억합니다.
* **중복 주문 차단**: 하루에 한 건만 주문을 넣을 수 있도록 클립보드 로컬스토리지 및 구글시트 서버단(DeviceId 기준)에서 이중으로 동일 일자 중복 주문을 차단합니다.
* **수령 방식 및 배달지 입력**: "포장 수령" 또는 "배달 서비스 이용"을 선택할 수 있으며, 배송지(기본값: *사무실 원탁*)가 비어있으면 주문이 접수되지 않습니다.
* **오늘의 배달팀 표시**: 관리자가 지정한 "오늘의 배달팀(배달 담당, 상품 준비 담당)" 정보와 한마디 메시지가 게스트 메인 화면에 생동감 있게 표시됩니다.
* **게스트 후기 팝업 뷰어**: 게스트들이 남긴 후기를 슬라이드 방식(이전/다음 버튼)으로 확대하여 볼 수 있는 모바일 최적화 후기 뷰어 모달을 탑재했습니다.

### 3) 주방 및 운영 화면 (`kitchen.html`)
* **3열 대기 주문 그룹화**: 들어온 주문을 **`🥡 키오스크 주문`**, **`📱 배달왔삼 포장`**, **`🛵 배달왔삼 배달`**의 3가지 전용 열(Column)로 명확히 분리하여 렌더링합니다. PC에서는 3열 배치로, 모바일에서는 1열 종배치로 자동 반응합니다.
* **열(컬럼)별 독립형 일괄 처리**: 각 컬럼 헤더 바로 아래에 개별 일괄 처리 바가 내장되어, 특정 열에 있는 주문들만 `선택 제공`하거나 `모두 제공`할 수 있습니다.
* **실시간 집계 보드**: 아직 만들어지지 않은 대기 중인 간식들의 수량을 실시간 종류별 합산하여 주방장에게 우선적으로 꺼내올 간식 목록을 시각화합니다.
* **오늘의 운영 결과 요약 모달**:
  * 온데만드 방식으로 터치 시에만 데이터를 계산하여 구글시트 API 및 대역폭 부하를 최소화합니다.
  * 총 주문 수, 총 차감 크레딧, 키오스크 주문/배달왔삼 포장/배달왔삼 배달 건수, 완료/취소 건수 및 후기 API를 연동하여 사진 후기율과 후기 태그 빈도를 집계합니다.
  * **원클릭 복사**: 메신저나 운영 보고서에 즉시 붙여넣을 수 있도록 깔끔한 텍스트 템플릿으로 복사되는 기능을 탑재했습니다.

### 4) 빌지 인쇄 및 체크리스트 (`print-bills.html`)
* **자동 영수증 인쇄**: 주방 접수 시 인쇄 화면을 통해 개별 빌지를 일괄 인쇄합니다.
* **배송지별 배달 체크리스트**: 배달을 가야 하는 배달 주문들을 배송지(예: *주간보호실 A*, *사무실 원탁*) 기준으로 가나다순 그룹화하여 테이블 형식의 통합 체크리스트를 생성합니다.
* **인쇄 레이아웃 제어**: 개별 빌지 카드의 출력이 끝난 후 CSS Page Break 규칙(`break-before: page`)을 적용해 **새 A4 용지 페이지에 체크리스트만 단독으로 인쇄**되도록 격리하여 용지 낭비와 레이아웃 깨짐을 방지합니다.

### 5) 후기 관리 및 설정 (`reviews.html`, `admin.html`)
* **완전 분리된 아키텍처**: 관리자 설정(`admin.html`), 주문/주방 처리(`kitchen.html`), 후기 모니터링(`reviews.html`) 화면을 완전히 분리하여 API 호출 충돌과 속도 저하를 완벽히 차단했습니다.
* **화면 갱신 일시정지**: 관리자가 팝업 모달을 열고 크레딧이나 간식 정보를 수정하는 동안에는 30초 자동 새로고침 타이머가 일시정지되어, 작성 중인 데이터가 날아가는 현상을 방지합니다.
* **후기 검수 및 사진 조회**: 후기 테이블에서 후기 사진 썸네일을 제공하며, 클릭 시 고해상도 전체 사진을 모니터링하고 노출 여부(공개/비공개)를 안전하게 관리할 수 있습니다.

### 6) PWA 및 오프라인 안심 시스템
* **전면 오프라인 오버레이**: 브라우저 기본 에러창 대신 따뜻한 파스텔톤 배경과 위아래로 움직이는 캐릭터 일러스트를 사용한 전면 대기 오버레이를 제공합니다. "인터넷이 다시 연결되기를 기다리고 있습니다." 문구와 함께 불안감을 낮추는 TTS 안심 멘트를 송출합니다.
* **독립형 멀티 PWA**: 4가지 모드(이용자용 키오스크, 관리자용 대시보드, 주방 모니터링, 대기실 전광판 등)가 각각 개별 앱처럼 단독 홈 화면 설치가 가능하도록 전용 PWA 매니페스트 파일들을 다원화했습니다.
  - `manifest-kiosk.json`, `manifest-admin.json`, `manifest-kitchen.json`, `manifest-board.json`, `manifest-guest.json`, `manifest-reviews.json`

## 📊 스프레드시트 구조 (Google Sheets DB)

시스템을 운영하려면 아래 7개 탭(시트)으로 구성된 구글 스프레드시트가 필요합니다.

1. **`이용자목록`**: `userId` (고유ID), `nickname` (별명), `credit` (잔액), `useYn` (활성화 여부: Y/N), `imageUrl` (사용자 사진 Drive ID)
2. **`간식목록`**: `snackId` (고유ID), `name` (간식명), `point` (가격), `imageUrl` (간식 사진 Drive ID), `saleYn` (판매여부: Y/N), `stock` (재고량), `displayOrder` (정렬순서), `target` (노출대상: user/guest)
3. **`주문내역`**: `timestamp` (주문 일시), `orderNo` (주문번호), `userId`, `nickname`, `snackId`, `snackName`, `quantity` (주문수량), `point` (차감포인트), `servedYn` (N:접수, P:준비, R:출발/완료, Y:제공완료, C:취소), `cancelTimestamp`, `orderToken`, `deliveryType` (pickup/delivery), `deliveryFee` (배달료), `totalCredit` (사용 후 잔액), `reviewed` (후기여부), `deliveryPlace` (배달 목적지), `guestDeviceId`, `authProvider`, `guestKey`
4. **`관리자로그`**: 관리자 로그인 정보 및 재고/크레딧 수정 로그 추적
5. **`운영설정`**: `key` (설정항목), `value` (설정값) - 게스트 운영 상태(open/closed), 남은 시간, 기본 배달 정보 및 오늘의 배달팀 데이터 관리
6. **`후기내역`**: `id`, `orderNo`, `nickname`, `rating`, `tags` (선택된 태그), `comment` (한마디), `imageUrl` (후기 사진), `createdAt`, `useYn` (후기 노출 여부: Y/N)
7. **`게스트프로필`**: `guestKey`, `displayName`, `deliveryPlace`, `updatedAt` - 카카오 연결 사용자가 선택 저장에 동의한 경우에만 마지막 주문표시명과 배송지를 보관합니다.

## 🛠️ 설치 및 로컬 개발

1. **로컬 실행**
   - 본 프로젝트는 순수 정적 웹페이지(HTML, CSS, JS)이므로 별도의 웹 서버나 빌드가 필요 없습니다.
   - VS Code의 **Live Server** 익스텐션을 활성화하거나, 터미널에서 `python -m http.server 8081` 명령어를 입력해 가볍게 로컬 서버를 구동할 수 있습니다.
   - 네트워크 연결이 닫혀 있거나 구글 Apps Script 주소가 배포 전일 때는 `js/config.js` 파일 내 `USE_MOCK = true;`로 설정하여 미리 탑재된 모형 데이터로 테스트할 수 있습니다.

2. **Apps Script 백엔드 구축**
   - 구글 스프레드시트를 하나 생성합니다.
   - 상단 메뉴의 `확장 프로그램` -> `Apps Script`를 클릭합니다.
   - [google-apps-script.md](./google-apps-script.md) 파일에 백업된 JavaScript 코드를 복사하여 스크립트 에디터에 붙여넣습니다.
   - `새 배포`를 진행하여 발급받은 웹앱 URL을 `js/config.js`의 `API_URL` 값에 설정합니다.
   - 카카오 선택 로그인을 운영하려면 Apps Script 프로젝트 속성에 `KAKAO_REST_API_KEY`, `KAKAO_GUEST_KEY_SALT`를 저장합니다. 카카오 콘솔에서 Client Secret을 켠 경우에만 `KAKAO_CLIENT_SECRET`도 저장합니다.
   - 카카오 Redirect URI는 GAS 주소가 아니라 실제 사용자가 여는 정적 앱의 `guest.html` 주소로 등록합니다.

3. **PWA 독립 구동 확인**
   - 브라우저 개발자 도구(F12)의 Application 탭에서 `service-worker.js`와 각 `manifest-*.json` 파일이 충돌 없이 등록되었는지 확인합니다.
   - 기기 화면 크기(반응형 및 모바일 뷰)와 오프라인 감지 오버레이가 정상적으로 뜨는지 확인해 보십시오.
