# 간식 키오스크

주간보호센터 발달장애인 이용자가 태블릿이나 스마트폰에서 자기 이름을 고르고, 원하는 간식을 선택해 크레딧으로 주문할 수 있는 PWA 키오스크입니다. 관리자는 별도 관리자 화면에서 주문 제공 여부, 이용자 크레딧, 간식 재고를 관리합니다.

---

## 주요 기능

### 이용자 화면
- **큰 터치 영역**: 이름 카드, 간식 카드, 주문 버튼을 크게 배치해 터치 실수를 줄였습니다.
- **사진/아바타 표시**: 이용자 사진이 있으면 표시하고, 없으면 별명 첫 글자 아바타를 보여줍니다.
- **간식 이미지 선택**: 작은 `+` 버튼뿐 아니라 간식 이미지 영역을 눌러도 수량이 1개씩 늘어납니다.
- **주문 확인 이미지**: 주문 확인 화면에서 선택한 간식의 이미지 또는 대체 이모지를 함께 보여줍니다.
- **햅틱/효과음 피드백**: 버튼, 카드, 간식 선택 시 진동과 Web Audio API 기반 효과음을 제공합니다.
- **음성 안내(TTS)**: 토글을 켜면 이용자 선택과 간식 추가/제거 내용을 한국어 음성으로 안내합니다.
- **품절/한도 가드**: 재고가 0인 간식은 선택할 수 없고, 보유 크레딧이나 재고를 초과한 주문은 차단합니다.

### 관리자 화면
- **주문 처리 현황**: 오늘 주문을 제공 대기/제공 완료로 분리해 보여줍니다.
- **제공 완료/되돌리기**: `servedYn` 값을 스프레드시트에 실시간 반영해 제공 상태를 관리합니다.
- **미제공 간식 총량**: 아직 제공하지 않은 간식을 종류별로 합산해 준비할 수 있게 보여줍니다.
- **크레딧 관리**: 이용자별 크레딧을 빠른 버튼 또는 직접 입력으로 변경합니다.
- **목표 크레딧 설정**: 테스트나 정산 보정이 필요할 때 `0`, `5`, `10`, `20` 또는 직접 입력값으로 보유 크레딧을 바로 맞춥니다.
- **이용자 추가**: 신규 이용자를 추가하고, 이용자를 활성/비활성 그룹으로 나누어 관리합니다. 완전 삭제는 프론트에서 제공하지 않습니다.
- **간식/재고 관리**: 재고 조정, 품절 처리, 신규 간식 등록을 관리자 화면에서 처리합니다.
- **간식 표시 관리**: 간식을 `판매중` 또는 `숨김` 상태로 전환해 키오스크 노출 여부를 관리합니다.
- **이미지 자동 업로드 (API 지원)**: 프론트 UI 상의 사진 업로드 버튼은 회사 정책상 제외하였으나, Apps Script 백엔드 연동 API(`action: 'uploadImage'`)는 여전히 활성화되어 있어 언제든 수신/업로드 처리가 가능합니다.
- **등록 중복 방지**: 구글시트 API 지연에 따른 중복 추가를 방지하기 위해 이용자/간식 등록 시 완료 전까지 버튼을 자동으로 비활성화 처리합니다.
- **CSV 다운로드**: 오늘 주문 내역을 CSV로 내려받아 정산이나 기록에 활용할 수 있습니다.
- **관리자 변경 로그**: 크레딧, 이용자 상태, 간식 상태, 재고 변경 등 관리자 작업을 `관리자로그` 시트에 기록합니다.
- **변경 메모**: 관리자 화면의 메모 입력값을 다음 변경 로그에 함께 남깁니다.
- **자동 새로고침 일시정지**: 관리자 화면의 30초 자동 갱신을 잠시 멈출 수 있습니다.
- **신규 주문 알림**: 자동 새로고침 중 새 주문이 감지되면 알림음을 재생합니다.
- **관리자 변경 API 보호**: `ADMIN_TOKEN` 기반으로 크레딧/재고/제공상태 변경 요청을 보호합니다.

### PWA 및 연동
- **홈 화면 앱 설치**: Android Chrome/Edge, iOS Safari에서 홈 화면에 추가해 독립 앱처럼 실행할 수 있습니다.
- **정적 파일 캐싱**: 서비스 워커가 HTML/CSS/JS/아이콘을 캐싱해 로딩 속도를 높입니다.
- **Google Apps Script API 연동**: 구글 스프레드시트를 이용자, 간식, 주문 DB로 사용합니다.
- **Google Drive 이미지 변환**: 드라이브 공유 링크나 파일 ID를 브라우저 표시 가능한 썸네일 URL로 자동 변환합니다.
- **동시 주문 잠금**: Apps Script의 `LockService`로 여러 기기 동시 주문 시 재고/크레딧 경합을 줄입니다.
- **표시값 안전 처리**: 스프레드시트에서 들어오는 이름, 이미지 주소, 주문 항목명은 HTML 표시 전 escape 처리합니다.

## 파일 구조

```
├── index.html            # 이용자 선택 화면
├── menu.html             # 간식 선택 화면
├── confirm.html          # 주문 확인 화면
├── complete.html         # 주문 완료 화면
├── admin.html            # 관리자 대시보드
├── google-apps-script.md # Google Apps Script 최종 코드 및 적용 방법
├── handoff.md            # 남은 점검/인수인계 메모
├── manifest.json         # PWA 마니페스트
├── service-worker.js     # 정적 파일 캐싱 및 API 요청 우회
├── css/
│   └── style.css
├── icons/
│   ├── icon-192.png
│   └── icon-512.png
└── js/
    ├── config.js         # API URL, Mock 데이터, API 헬퍼
    └── app.js            # 공통 상태, 이미지 URL 변환, 피드백, PWA 등록
```

## 스프레드시트 구조

### 이용자목록
- **이용자ID** (`userId`): 고유 식별 코드
- **별명** (`nickname`): 화면에 표시할 쉬운 이름
- **크레딧** (`credit`): 사용 가능한 크레딧
- **사용여부** (`useYn`): `Y`, `TRUE`, `사용`, `O`, `예` 등 활성 값
- **사진URL** (`imageUrl`): 이용자 사진 URL 또는 Google Drive 파일 ID

### 간식목록
- **간식ID** (`snackId`): 고유 식별 코드
- **이름** (`name`): 간식 이름
- **포인트** (`point`): 주문 시 차감할 크레딧
- **사진URL** (`imageUrl`): 간식 사진 URL 또는 Google Drive 파일 ID
- **판매여부** (`saleYn`): `Y`, `TRUE`, `판매`, `O`, `예` 등 활성 값
- **재고** (`stock`): 잔여 수량, `0`이면 품절

### 주문내역
- **주문시간** (`timestamp`)
- **주문번호** (`orderId` 또는 `orderNo`)
- **이용자ID** (`userId`)
- **별명** (`nickname`)
- **간식ID** (`snackId`)
- **간식명** (`snackName`)
- **수량** (`quantity`)
- **차감포인트** (`point`)
- **제공여부** (`servedYn`): 기본값 `N`, 제공 완료 시 `Y`

## 실행 및 배포

1. **로컬 실행**
   - 정적 파일 프로젝트이므로 VS Code Live Server 또는 `python -m http.server 8080`으로 실행할 수 있습니다.
   - `js/config.js`의 `USE_MOCK`을 `true`로 바꾸면 로컬 Mock 데이터로 테스트합니다.

2. **Apps Script 적용**
   - [google-apps-script.md](./google-apps-script.md)의 최종 코드를 Apps Script 편집기에 붙여넣습니다.
   - 프로젝트 설정의 스크립트 속성에 `ADMIN_TOKEN`을 등록합니다.
   - 웹앱으로 배포하고 발급된 URL을 `js/config.js`의 `API_URL`에 넣습니다.
   - 이용자 추가/비활성화 기능을 쓰려면 최신 Apps Script 코드가 배포되어 있어야 합니다.

3. **PWA 확인**
   - 브라우저 개발자 도구의 Application 탭에서 Manifest와 Service Worker가 정상 등록됐는지 확인합니다.
   - Android Chrome/Edge: 배포 주소 접속 후 **앱 설치** 또는 **홈 화면에 추가**.
   - iPhone/iPad Safari: 공유 버튼에서 **홈 화면에 추가**.
   - 홈 화면 아이콘 실행 시 주소창 없는 독립 앱 화면인지 확인합니다.
