# 배달왔삼 프로젝트 핸드오프

이 파일은 프로젝트 문서의 **유일한 입구**입니다. 앞으로도 "핸드오프를 봐줘"라고 하면 이 파일부터 확인하면 됩니다. 세부 정보가 필요할 때만 아래 연결 문서를 추가로 읽습니다.

## 현재 상태

- **치명 이슈**: 없음
- **문서 기준일**: 2026-07-12
- **운영 DB 검증**: `주간보호 매점DB (2).xlsx`의 실제 시트 10개와 1행 헤더를 대조했으며, 신규 `이용신청` 시트는 GAS 배포 후 생성·검증 대기
- **GAS 구조**: `gas/*.gs` 기능별 분리 완료, 같은 Apps Script 프로젝트 안에서 함께 배포
- **정적 캐시**: `service-worker.js`의 현재 캐시 버전은 `kiosk-cache-v197`
- **최근 확인**: 기존 관리자·주방·후기와 주문 안정화 작업에 더해 **배달왔삼 이용 신청 공개 페이지, 관리자 신청 탭, 신규 `이용신청` 시트 계약, 멱등 접수·중복 연락처 차단·30일 만료 익명화, 1차 시범 신청 5명 서버 제한을 구현했습니다.** 아기곰 마스코트(달곰이) 일러스트와 반응형 디자인, 예전 신청 설정 기본값 보정도 완료했으며 GAS 배포·운영 검증을 기다립니다.

## 상담 후 다음 모바일 개선 우선순위

1. **M-P1 - 관리자 로그인 게이트 검토·구현 (구현 및 배포 후 검증 완료)**
   - 관리자·주방·후기 화면 진입 시 로그인 게이트를 먼저 보여주고, 인증 성공 후 본문을 표시하도록 구현했습니다.
   - 기존 `AdminAuth`를 재사용하고, 인증 성공 이후에만 초기 데이터 조회가 실행되도록 연결했습니다.
   - 로그인 후 기존 인증 바는 숨기며, 기능 행의 열쇠 버튼으로 잠글 수 있습니다.
2. **M-P2 - 게스트 주문 운영 패널 모바일 재배치 및 운영 설정 접기 (구현 완료, 추가 수동 검증 대기)**
   - 마감 예정·남은 시간과 기본 크레딧·배달비를 2열로 정리하고, 기본 배달지는 전체 폭으로 배치했습니다.
   - 배달팀 설정과 운영 시간 버튼을 모바일에서 읽기 쉬운 그룹으로 재배치했습니다.
   - 자동 새로고침 중 입력 중인 값을 덮어쓰지 않는 현재 보호 동작은 유지했습니다.
   - 모바일 기본 크레딧·배달비 입력칸의 침범 방지, 기본 크레딧 라벨 `기본 🪙`, 배달팀 설정의 `표시`·`제목:` 여백 및 메시지 입력 표시, `⏱ 직접 열기:` 한 줄 표시를 보완했습니다.
   - 운영 상태·마감 버튼은 항상 표시하고 상세 설정은 기본 접힘 `⚙ 운영 설정` 패널로 분리했습니다.
   - **기존 레이아웃 검증 완료, 추가 검증 필요**: PC·390px·320px에서 새 접기·펴기 구조와 설정 저장을 확인합니다.
3. **M-P3 - 주방 상단 요약 카드 문구·값 분리 (구현 및 수동 검증 완료)**
   - `오늘 누적 크레딧`·`오늘 총 주문`으로 제목을 줄이고, 아래에 `🪙 숫자`·`숫자건`을 배치했습니다.
   - 현재 2열 카드 구조와 집계 의미는 유지합니다.
   - **수동 검증 완료**: PC·390px·320px에서 제목 줄바꿈과 실제 값 갱신을 확인했습니다.
4. **M-P4 - 주방 화면 주문·통계·재고 탭 분리 (구현 및 수동 검증 완료)**
   - 주문·통계·재고를 PC·모바일 공통 탭으로 분리했습니다.
   - 기존 API·자동 새로고침·주문 처리 구조는 유지하고 화면 표시 영역만 전환합니다.
   - **수동 검증 완료**: 세 화면 크기에서 탭 전환, 주문 처리, 통계·재고 표시와 가로 넘침 없음을 확인했습니다.

## 다음 우선순위

1. **P13 - 배달왔삼 이용 신청·승인 관리 (디자인 고도화 완료, GAS 배포·운영 검증 대기)**
   - 공개 신청 페이지와 관리자 신청 탭, `이용신청` 19열, 멱등 접수, 상태 처리, 30일 만료 익명화를 구현했습니다.
   - 신규 아기곰 마스코트(**달곰이**) 3단 일러스트 연동, 헤더 이중 포인트 채색, PC/모바일 정렬 개선 및 캐릭터 머리 크롭 방지 스타일 적용을 완료했습니다.
   - 운영 DB 백업 후 GAS 반영, `setupGuestApplicationSheet()` 실행, 신청 마감 상태 검증, 개인정보 문구 기관 검토가 남아 있습니다.
2. **P14 - 배달왔삼 1차 시범 신청 5명 제한 (구현 완료, GAS 배포·수동 검증 대기)**
   - `PENDING + APPROVED`를 5명 정원으로 계산하고 반려·중지 시 자리를 다시 열며, 동일 `requestId` 재전송은 기존 신청 결과를 반환합니다.
   - 정원이 차면 서버가 잠금 안에서 접수를 거절하고 개인정보를 저장하지 않습니다. 대기열과 신청자 상태 조회 페이지는 만들지 않고 승인·반려 결과를 기관에서 직접 연락합니다.
3. **P1 - 비회원 게스트 주문표시명 하루 1회 정책 (구현 및 수동 검증 완료)**
   - 주문 성공 후 같은 기기·같은 날짜에는 표시명을 읽기 전용으로 유지하고, 다음 날 새 표시명을 입력할 수 있게 했습니다.
4. **P2 - 관리 화면 전환 탭과 기능 버튼 분리 (구현 및 수동 검증 완료)**
   - 관리자·주방·후기 화면 전환을 고유 색상의 전역 탭으로 분리하고, 알림·운영 도구·외부 화면 등 기능 버튼과 시각적 역할을 나눴습니다.
5. **P3 - 주방 간식 재고 현황 복구 및 그룹 표시 (구현 및 수동 검증 완료)**
   - 게스트·일반 그룹으로 나누어 간식명과 재고만 표시하고, 재고 0은 하이라이트로 구분합니다.
   - `getSnacks` 조회와 화면 렌더링을 연결하되, 최종 주문 단계의 서버 재고 검증은 그대로 유지합니다.
6. **P4 - 외부 화면 빠른 버튼 색상 실험 (구현 및 수동 검증 완료)**
   - 관리자·주방·후기 화면 모두 좌측에 일반 키오스크·배달왔삼·전광판 링크를 배치했습니다.
   - 일반 키오스크는 주황색, 배달왔삼은 청록색, 전광판은 보라색으로 구분합니다.
   - 빌지 인쇄는 주방 화면에만 우측 독립 버튼으로 유지하고, 관리자·후기 화면에는 노출하지 않습니다.
   - 세 화면의 링크 이동과 모바일 배치를 수동 검증했습니다.
7. **P5 - 주방 재고 게이지 표시 (구현 및 수동 검증 완료)**
   - 카드 내부에 잔여 재고량 백분율만큼 연한 단색 게이지 배경(10개 이상 초록, 4~9개 노랑, 1~3개 빨강, 0개 미적용)을 채워 가시성을 확보했습니다.
   - 정확한 재고 숫자와 간식명은 게이지 텍스트 레이어로 얹어 묻힘 없이 표시합니다.
8. **P6 - 후기 목록 `더 보기` 방식 (구현 및 수동 검증 완료)**
   - 기존 후기 조회 결과를 공개/숨김 필터 적용 후 20개씩 표시하고 `후기 더 보기`로 다음 목록을 추가합니다.
   - GAS API는 변경하지 않았으며, 카드 상세·이전·다음 인덱스와 필터 동작을 유지합니다.
   - **수동 검증 완료**: 20개 초과 후기의 추가 표시, 필터, 상세·이전·다음, 답글·공개 상태 변경을 확인했습니다.
9. **P7 - 후기 참여 정보 표시 검토**
   - 후기 상세 화면의 참여 정보 표시 필요성과 개인정보 영향을 검토합니다.
10. **P8 - 미사용 CSS 및 잔여 코드 정리 (낮은 우선순위, 보류)**
   - 화면 개편 뒤 참조되지 않는 CSS 선택자, 중복 인라인 스타일, 호출되지 않는 JavaScript를 찾아 정리하는 유지보수 작업입니다.
   - 회귀 위험이 있어 기능 안정화 이후 작은 단위로 진행합니다.
11. **운영 모니터링**
   - GAS 429/503 또는 락 대기 오류 재발 여부와 특정 간식 주문 가능 상태의 일회성 불일치를 관찰합니다.
12. **P9 - 관리 화면 PWA 아이콘 3종 분리 (폐기)**
   - 공통 관리자 이미지 하나를 관리자·주방·후기 화면에서 함께 사용하는 현재 방식을 유지하기로 결정했습니다.
   - 화면별 별도 아이콘 생성·교체는 중복 에셋과 관리 부담이 생기므로 진행하지 않습니다.

### 완료된 검토

- **`placeOrder` 트랜잭션 정합성**: `LockService`와 사전 검증은 유지하되, Google Sheets의 비원자적 다중 쓰기 때문에 부분 실패 위험이 있음을 확인했습니다. 실제 불일치가 관찰되면 복구 구조를 재검토합니다.

- **화면 전환 탭과 기능 버튼 분리**: 관리자·주방·후기 화면 전환을 고유 색상의 전역 탭으로 분리하고, 알림·운영 도구·외부 화면 등 현재 화면의 기능 버튼과 시각적 역할을 나눕니다.

상세 상태와 착수 조건은 [현재 작업과 우선순위](docs/handoff/active-work.md)에 있습니다.

## AI 작업 시작 규칙

1. 이 파일과 `active-work.md`를 먼저 읽습니다.
2. 작업 영역에 따라 `architecture.md`, `decisions.md`, `database-schema.md`, `verification.md`만 추가로 읽습니다.
3. 실제 DB 구조를 추측하지 않습니다. 구조 변경 전에는 최신 DB 사본 또는 운영 점검 결과를 확인합니다.
4. 운영 중인 시트 열을 코드 가정에 맞춰 임의로 이동하거나 삭제하지 않습니다.
5. GAS를 수정하면 결합 구문 검사 후 새 버전을 배포하고, 정적 파일을 수정하면 서비스 워커 캐시 버전 상향 필요성을 확인합니다.
6. 완료된 작업의 짧은 결과는 `work-log.md`에 남기고, 긴 분석 원문은 아카이브로 보냅니다. `handoff.md`에는 현재 상태와 링크만 유지합니다.
7. 모든 화면 변경은 [모바일 친화적 개발 원칙](docs/handoff/decisions.md#모바일-친화적-개발-원칙)을 먼저 확인하고, PC·390px·320px 회귀 검증 범위를 함께 정합니다.
