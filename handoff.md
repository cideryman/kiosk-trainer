# 배달왔삼 프로젝트 핸드오프

이 파일은 프로젝트 문서의 **유일한 입구**입니다. 앞으로도 "핸드오프를 봐줘"라고 하면 이 파일부터 확인하면 됩니다. 세부 정보가 필요할 때만 아래 연결 문서를 추가로 읽습니다.

## 현재 상태

- **치명 이슈**: 없음
- **문서 기준일**: 2026-07-10
- **운영 DB 검증**: `주간보호 매점DB (2).xlsx`의 실제 시트 10개와 1행 헤더를 다시 대조함
- **GAS 구조**: `gas/*.gs` 기능별 분리 완료, 같은 Apps Script 프로젝트 안에서 함께 배포
- **정적 캐시**: `service-worker.js`의 현재 캐시 버전은 `kiosk-cache-v147`
- **최근 확인**: 관리자 잠금 해제 UI 교체 및 GAS 키 로그 보호 적용, 관리 화면 하위 메뉴 배타 동작, 배달왔삼 실제 화면 연결, 마감 중 읽기 전용 메뉴 보기, 예약 종료 주문 5분 유예, 관리자·후기 화면 1200px 확대 및 주방 화면 1400px 유지(하이브리드 조율), 주방 재고량 표시 게이지 UI 구현 및 검증 완료, 관리자 화면 간식 관리 테이블 중복 태그 제거 및 슬림 재고 게이지 적용 완료, 관리자 화면 내 이용자 및 간식 테이블 5열 단일화(상태 배지 제거, 게이지+수량 병합) 및 버튼 줄바꿈 버그 해결 완료, 관리자 화면 내 이용자 탭 검색란 제거 및 상단 컨트롤러 UI 통일(검색 기능 완전 폐지) 완료, 간식 관리 테이블 헤더 명칭 단축('재고')으로 줄바꿈 해소 완료, 후기 관리 화면(reviews.html)을 표 형식에서 카드 뷰 레이아웃(주문번호는 호버 툴팁 처리)으로 개선 완료

## 다음 우선순위

1. **P1 - 비회원 게스트 주문표시명 하루 1회 정책 (구현 완료, 날짜 전환 검증 대기)**
   - 주문 성공 후 같은 기기·같은 날짜에는 표시명을 읽기 전용으로 유지하고, 다음 날 새 표시명을 입력할 수 있게 했습니다.
2. **P2 - 관리 화면 전환 탭과 기능 버튼 분리 (구현 및 수동 검증 완료)**
   - 관리자·주방·후기 화면 전환을 고유 색상의 전역 탭으로 분리하고, 알림·운영 도구·외부 화면 등 기능 버튼과 시각적 역할을 나눴습니다.
3. **P3 - 주방 간식 재고 현황 복구 및 그룹 표시 (구현 및 수동 검증 완료)**
   - 게스트·일반 그룹으로 나누어 간식명과 재고만 표시하고, 재고 0은 하이라이트로 구분합니다.
   - `getSnacks` 조회와 화면 렌더링을 연결하되, 최종 주문 단계의 서버 재고 검증은 그대로 유지합니다.
4. **P4 - 외부 화면 빠른 버튼 색상 실험 (부분 구현, 관리자·후기 적용 필요)**
   - 현재 주방 화면에만 일반 키오스크·배달왔삼·전광판 링크를 좌측에 적용했습니다.
   - 관리자·후기 화면은 아직 기존 `외부 화면` 하위 메뉴를 사용하므로 세 화면의 UI가 일치하지 않습니다.
   - 관리자·후기 화면에도 같은 링크 구조를 적용하고 빌지 인쇄 위치를 통일한 뒤 다시 검증합니다.
5. **P5 - 주방 재고 게이지 표시 (구현 및 수동 검증 완료)**
   - 카드 내부에 잔여 재고량 백분율만큼 연한 단색 게이지 배경(10개 이상 초록, 4~9개 노랑, 1~3개 빨강, 0개 미적용)을 채워 가시성을 확보했습니다.
   - 정확한 재고 숫자와 간식명은 게이지 텍스트 레이어로 얹어 묻힘 없이 표시합니다.
6. **P6 - 후기 참여 정보 표시 검토 (다음 작업)**
   - 후기 상세 화면의 참여 정보 표시 필요성과 개인정보 영향을 검토합니다.
7. **P7 - 미사용 CSS 및 잔여 코드 정리**
   - 화면 분리 과정에서 남은 미사용 선택자와 비활성 코드를 점진적으로 정리합니다.
8. **운영 모니터링**
   - GAS 429/503 또는 락 대기 오류 재발 여부와 특정 간식 주문 가능 상태의 일회성 불일치를 관찰합니다.

### 완료된 검토

- **`placeOrder` 트랜잭션 정합성**: `LockService`와 사전 검증은 유지하되, Google Sheets의 비원자적 다중 쓰기 때문에 부분 실패 위험이 있음을 확인했습니다. 실제 불일치가 관찰되면 복구 구조를 재검토합니다.

- **화면 전환 탭과 기능 버튼 분리**: 관리자·주방·후기 화면 전환을 고유 색상의 전역 탭으로 분리하고, 알림·운영 도구·외부 화면 등 현재 화면의 기능 버튼과 시각적 역할을 나눕니다.

상세 상태와 착수 조건은 [현재 작업과 우선순위](docs/handoff/active-work.md)에 있습니다.

## AI 작업 시작 규칙

1. 이 파일과 `active-work.md`를 먼저 읽습니다.
2. 작업 영역에 따라 `architecture.md`, `decisions.md`, `database-schema.md`, `verification.md`만 추가로 읽습니다.
3. 실제 DB 구조를 추측하지 않습니다. 구조 변경 전에는 최신 DB 사본 또는 운영 점검 결과를 확인합니다.
4. 운영 중인 시트 열을 코드 가정에 맞춰 임의로 이동하거나 삭제하지 않습니다.
5. GAS를 수정하면 결합 구문 검사 후 새 버전을 배포하고, 정적 파일을 수정하면 서비스 워커 캐시 버전 상향 필요성을 확인합니다.
6. 완료된 작업의 짧은 결과는 `work-log.md`에 남기고, 긴 분석 원문은 아카이브로 보냅니다. `handoff.md`에는 현재 상태와 링크만 유지합니다.

- **에셋 추가/교체**: `asset-sources.md` 갱신
