# 간식 키오스크 인수인계

이 문서는 다음 작업자가 이미 완료된 기능을 다시 구현하지 않고, 남은 확인 작업만 빠르게 볼 수 있도록 정리한 메모입니다. 완료 기능의 상세 설명은 `README.md`에 통합되어 있습니다.

---

## 현재 상태

- 이용자 주문 흐름, 관리자 화면, PWA 설정, Google Apps Script 연동은 구현 완료.
- **주문 취소 및 자동 환불/재고 복구 기능 구현 완료**: 
  - 관리자 화면의 "간식 챙겨주기 (제공 대기)" 목록 카드에 **[주문 취소]** 버튼을 추가했습니다.
  - 주문을 취소하면 스프레드시트의 해당 주문 건 `servedYn` 상태가 `'C'`(Cancelled)로 변경되며, 10번째 열(J열, `cancelTimestamp`)에 취소 시간이 기록됩니다.
  - 이와 동시에 **유저의 사용 크레딧이 자동 환불**되고, **해당 간식의 재고 수량이 자동 원상복구**됩니다.
- **주민 메인 화면 실시간 취소 알림 배너 구현 완료**:
  - 주민 메인 화면(`index.html`)이 5초마다 실시간으로 오늘 주문 내역을 조회(폴링)합니다.
  - 최근 3분 이내에 취소된 신규 취소 건이 발견되면, 화면 상단에 `📢 --님 님의 주문이 취소되었습니다.` 배너를 **60초 동안 노출**한 뒤 자동으로 숨깁니다.
  - 기기 새로고침 시 이미 이전에 취소된 구버전 알림이 반복해서 노출되는 문제를 방지하기 위해, 화면 진입 시 이미 취소된 주문번호들을 `dismissedCancellations` 캐시(로컬 스토리지)에 사전 등록하여 필터링합니다.
- **관리자 변경 API 보안 및 패스워드**:
  - 관리자 변경 API는 `ADMIN_TOKEN`으로 보호됩니다.
  - 현장 실사용 **관리자 비밀번호는 `Day6415`** 입니다. (세션 스토리지에 자동 저장되어 세션 내 유지됨)
- **정적 리소스 업데이트 적용을 위한 PWA 캐시 갱신 및 강력 새로고침 방법**:
  - 서비스 워커의 정적 캐시 충돌을 방지하기 위해 `CACHE_NAME`을 `kiosk-cache-v20`으로 갱신했습니다.
  - 코드 변경점이 제대로 보이지 않을 때는 브라우저의 서비스 워커 캐시를 완전히 비우고 새로운 스크립트를 로드하기 위해 **강력 새로고침 (Ctrl + Shift + R 또는 Ctrl + F5)** 또는 **F12 개발자 도구 > 새로고침 버튼 우클릭 > '캐시 비우기 및 강력 새로고침'**을 실행해 확인해야 합니다.

## 건드리지 않아도 되는 완료 항목

아래 항목들은 검증 완료되었거나 기본적으로 구현이 완료되어 있습니다.

- **LockService 배포 반영**: Apps Script에 `LockService.getScriptLock()` 코드가 정상 반영되어 동시 주문 경합을 차단합니다.
- **실사용 기기 PWA 설치**: Android/iOS 기기에서 홈 화면에 정상 추가되어 독립 앱 화면으로 실행됩니다.
- **센터 실사용 점검**: 
  - 이용자 선택 → 간식 선택 → 주문 확인 → 주문 완료 흐름 확인 완료.
  - 제공 대기 → 제공 완료 전환 및 되돌리기 확인 완료.
  - 관리자 화면의 실시간 알림음(신규 주문 / 주문 취소) 및 취소 알림 배너 작동 확인 완료.
  - 크레딧 조정, 재고 조정 및 품절 처리 확인 완료.
- **주문번호 발급 기능 구현 완료** (2026-06-09):
  - 이용자 메인 화면(`index.html`) 상단에 **주문번호 발급 토글**을 추가했습니다. 기본값은 켜짐(`true`)입니다.
  - 토글을 켜면, 주문 완료 시 `ORD-YYMMDD-001` 형식의 당일 순번 주문번호가 발급됩니다.
  - 주문 완료 화면(`complete.html`)에서 큰 글씨로 주문번호를 표시하고, TTS로 번호를 안내합니다.
  - 관리자 화면(`admin.html`)의 "제공 대기" 카드 및 "오늘 주문 내역" 표에도 주문번호가 앞에 표시됩니다.
  - Google Apps Script(`google-apps-script.md`)의 `placeOrder` 함수도 당일 순번 기반 주문번호를 생성하도록 업데이트했습니다. **Apps Script에 최신 코드를 붙여넣고 반드시 [새 배포]** 해야 실제 운영에서 순번 주문번호가 발급됩니다.
- **터치 성공률 개선 구현 완료** (2026-06-09):
  - `js/app.js`의 `AppState.bindCardTap()` 함수로 터치 지터 보정 로직을 공통화했습니다.
  - 이용자 선택 카드(`index.html`), 간식 카드 전체 및 +/- 버튼(`menu.html`)에 모두 적용되었습니다.
  - 터치 시작/끝 좌표 차이가 25px 이하이고 500ms 이내 탭으로 인정하며, 스크롤과의 충돌을 방지합니다.
