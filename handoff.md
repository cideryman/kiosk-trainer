# 간식 키오스크 인수인계

이 문서는 다음 작업자가 이미 완료된 기능을 다시 구현하지 않고, 남은 확인 작업만 빠르게 볼 수 있도록 정리한 메모입니다. 완료 기능의 상세 설명은 `README.md`에 통합되어 있습니다.

## 현재 상태

- 이용자 주문 흐름, 관리자 화면, PWA 설정, Google Apps Script 연동은 구현 완료.
- **주문 취소 및 자동 환불/재고 복구 기능 구현 완료**: 
  - 관리자 화면의 "간식 챙겨주기 (제공 대기)" 목록 카드에 **[주문 취소]** 버튼을 추가했습니다.
  - 주문을 취소하면 스프레드시트의 해당 주문 건 `servedYn` 상태가 `'C'`(Cancelled)로 변경되며, 10번째 열(J열, `cancelTimestamp`)에 취소 시간이 기록됩니다.
  - 이와 동시에 **유저의 사용 크레딧이 자동 환불**되고, **해당 간식의 재고 수량이 자동 원상복구**됩니다.
- **주민 메인 화면 실시간 취소 알림 배너 구현 완료**:
  - 주민 메인 화면(`index.html`)이 5초마다 실시간으로 오늘 주문 내역을 조회(폴링)합니다.
  - 최근 3분 이내에 취소된 신규 취소 건이 발견되면, 화면 상단에 `📢 --님 님의 주문이 취소되었습니다.` 배너를 **60초 동안 노출**한 뒤 자동으로 숨깁니다.
  - 기기 새로고침 시 이미 이전에 취소된 구버전 알림이 반복해서 노출되는 문제를 방지하기 위해, 화면 진입 시 이미 취소된 주문번호들을 `dismissedCancellations` 캐시(로컬 스토리지)에 사전 등록하여 필터링합니다.
- **관리자 변경 API 보안 및 패스워드**:
  - 관리자 변경 API는 `ADMIN_TOKEN`으로 보호됩니다.
  - 현장 실사용 **관리자 비밀번호는 `Day6415`** 입니다. (세션 스토리지에 자동 저장되어 세션 내 유지됨)
- **정적 리소스 업데이트 적용을 위한 PWA 캐시 갱신 및 강력 새로고침 방법**:
  - 서비스 워커의 정적 캐시 충돌을 방지하기 위해 `CACHE_NAME`을 `kiosk-cache-v18`로 갱신했습니다.
  - 코드 변경점이 제대로 보이지 않을 때는 브라우저의 서비스 워커 캐시를 완전히 비우고 새로운 스크립트를 로드하기 위해 **강력 새로고침 (Ctrl + Shift + R 또는 Ctrl + F5)** 또는 **F12 개발자 도구 > 새로고침 버튼 우클릭 > '캐시 비우기 및 강력 새로고침'**을 실행해 확인해야 합니다.

## 아직 확인해야 할 것

### 1. LockService 배포 반영 여부

`ADMIN_TOKEN`은 실제 웹앱에 반영된 것으로 확인됐지만, `LockService` 주문 잠금 코드는 외부 API 응답만으로 적용 여부를 확정하기 어렵습니다.

확인 방법:
- Apps Script 편집기에서 `placeOrder(data)` 함수 안에 `LockService.getScriptLock()`이 있는지 확인합니다.
- 없다면 `google-apps-script.md`의 최종 코드를 다시 붙여넣고 새 웹앱 배포를 진행합니다.

### 2. 실사용 기기 PWA 설치

각 태블릿/스마트폰에서 홈 화면 바로가기를 추가해야 합니다.

- Android Chrome/Edge: 배포 주소 접속 → **앱 설치** 또는 **홈 화면에 추가**
- iPhone/iPad Safari: 공유 버튼 → **홈 화면에 추가**
- 아이콘 이름은 `간식 키오스크`로 맞춥니다.
- 홈 화면 아이콘으로 실행했을 때 주소창이 없는 독립 앱 화면인지 확인합니다.

### 3. 센터 실사용 점검

현장에서 한 번만 확인하면 되는 항목입니다.

- 이용자 선택 → 간식 선택 → 주문 확인 → 주문 완료 흐름
- 관리자 화면의 새 주문 알림음
- 제공 대기 → 제공 완료 전환
- 제공 완료 → 되돌리기
- 크레딧 변경
- 재고 변경 및 품절 처리
- 신규 간식 등록
- 두 기기에서 같은 간식을 거의 동시에 주문했을 때 재고가 음수로 내려가지 않는지 확인

## 남은 개선 후보

현재 운영을 막는 문제는 아니지만, 추후 여유가 있으면 정리할 항목입니다.

### Apps Script 최신 코드 재배포

프론트에는 이용자 추가/활성화/비활성화, 간식 판매중/숨김, CSV 다운로드, 자동 새로고침 일시정지, 관리자 변경 메모, 그리고 **주문 취소 및 자동 환불/재고 복구(cancelOrder)** 기능이 들어갔습니다. `google-apps-script.md`에도 `addUser`, `updateUserActive`, `updateSnackSale`, `cancelOrder`, `관리자로그` 기록 코드가 모두 정의되어 있습니다. 실제 운영 웹앱에서 이 모든 신규 기능들을 사용하려면 Apps Script 편집기에 최신 코드를 붙여넣고 **반드시 [새 배포]를 진행**해야 합니다.

### 주문번호 호출 방식 검토

주문 완료 화면에 짧은 주문번호를 보여주고, 관리자가 "000번 음식 나왔습니다"처럼 부르는 방식은 아이디어로 보류합니다. 도입하려면 이용자가 번호를 기억/인지할 수 있는지, 관리자 화면에서 같은 번호가 크게 보이는지, 당일 번호 충돌이 없는지를 함께 설계해야 합니다.

### 터치 성공률 개선 검토

기기 자체의 터치 민감도를 웹 코드로 직접 높일 수는 없습니다. 대신 이용자가 버튼을 정확히 누르지 못해도 성공하도록 아래 방식으로 보정할 수 있습니다.

- 터치 영역 확대: 버튼 시각 크기보다 실제 클릭 영역을 더 크게 만들기 (`::before` 또는 padding/min-height 확대).
- 카드 전체 선택: 작은 버튼뿐 아니라 카드/이미지/이름 영역 전체를 선택 트리거로 쓰기.
- 터치 흔들림 보정: `touchstart` 좌표와 `touchend` 좌표 차이가 작으면 탭으로 인정하기.
- 길게 누름 허용: `pointerdown` 후 일정 시간 안에 떼지 않아도 1회 선택 처리하기.
- 중복 터치 방지: 같은 요소를 짧은 시간에 여러 번 눌러도 1회만 처리하거나, 반대로 `+` 버튼은 의도한 빠른 반복 입력만 허용하기.
- 스크롤/탭 충돌 완화: 터치 영역 안에서 아주 작은 움직임은 스크롤이 아니라 탭으로 처리하고, 큰 움직임만 스크롤로 인정하기.
- 핵심 화면 우선 적용: 이용자 선택 카드, 간식 카드, `+/-` 버튼, 주문하기 버튼, 관리자 제공 완료 버튼 순서로 적용하면 효과가 큽니다.

주의: 너무 민감하게 만들면 실수 주문이 늘 수 있으므로, 주문 확정 화면은 현재처럼 한 번 더 확인하는 구조를 유지해야 합니다.
